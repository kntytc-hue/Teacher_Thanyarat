    Unit 03: Process Architecture & State Transitions

    หน่วยที่ 3 การจัดการโพรเซส

    1. ความแตกต่างระหว่าง Program, Process และ Thread

    ในโครงสร้างสถาปัตยกรรมระบบคอมพิวเตอร์ โปรแกรม (Program) จัดเป็นองค์ประกอบสถานะสแตติก (Passive Entity) หมายถึงชุดของข้อคำสั่ง (Instructions) ที่บันทึกอยู่บนระบบไฟล์คอมพิวเตอร์อย่างถาวร เช่น บนฮาร์ดดิสก์ หรือ SSD โดยที่ยังไม่มีการจัดสรรหน่วยความจำหรือรอบเวลาทำงานจากซีพียู

    ในขณะที่ โพรเซส (Process) คือออบเจกต์สถานะไดนามิก (Active Entity) เกิดขึ้นเมื่อโปรแกรมดังกล่าวถูกโหลดเข้าสู่หน่วยความจำหลัก (RAM) และกำลังทำงานอยู่ภายใต้การควบคุมของระบบปฏิบัติการ โดยแต่ละโพรเซสจะมีขอบเขตหน่วยความจำส่วนตัว (Address Space) ประกอบด้วย Text Section (เก็บโค้ดคำสั่ง), Data Section (เก็บตัวแปรโกลบอล), Heap (พื้นที่จองแรมแบบไดนามิก) และ Stack (เก็บตัวแปรท้องถิ่นและฟังก์ชันคอล)

    ส่วน เธรด (Thread) คือหน่วยย่อยที่สุดของการรันคำสั่งภายในโพรเซส (Lightweight Process) โพรเซสหนึ่งตัวสามารถแยกโครงสร้างการคำนวณออกเป็นหลายเธรดทำงานพร้อมกัน (Multi-threading) โดยทุกเธรดภายใต้โพรเซสแม่เดียวกันจะสามารถแบ่งปัน (Share) พื้นที่โค้ด ข้อมูล และไฟล์ร่วมกันได้ทันที ทำให้ใช้ทรัพยากรน้อยกว่าการสร้างโพรเซสใหม่แยกอิสระเป็นอย่างมาก

    2. โครงสร้างบล็อกควบคุมโพรเซส (Process Control Block: PCB)

    เพื่อรักษาสถานะและควบคุมดูแลทุกโพรเซสภายในระบบปฏิบัติการ Kernel จะสร้างโครงสร้างข้อมูลชุดหนึ่งขึ้นมาเรียกว่า Process Control Block (PCB) ทำหน้าที่เปรียบเสมือนสมุดบันทึกประวัติการทำงานประจำตัวของโพรเซสนั้นๆ ข้อมูลสำคัญภายใน PCB ประกอบด้วย:

    • Process ID (PID): หมายเลขรหัสจำนวนเต็มบวกที่ระบบสลักไว้ระบุตัวตนโพรเซสไม่ให้ซ้ำกัน
    • Process State: สถานะปัจจุบัน (เช่น Ready, Running, Waiting)
    • Program Counter (PC): พอยน์เตอร์ชี้พิกัดหน่วยความจำถาวรของคำสั่งถัดไปที่ต้องประมวลผลต่อ
    • CPU Registers: ค่าข้อมูลที่ค้างอยู่ในรีจิสเตอร์ของสถาปัตยกรรมซีพียู ซึ่งต้องบันทึกเก็บไว้เมื่อเกิดการสลับงาน
    • Memory Management Information: ตารางระบุขอบเขตพิกัดหน่วยความจำ เช่น Page Tables หรือ Segment Tables

    3. โครงสร้างคิวการจัดตารางเวลา (Scheduling Queue)

    เมื่อมีโพรเซสเข้าสู่ระบบพร้อมกันเป็นจำนวนมาก ระบบปฏิบัติการจะนำโพรเซส (ในรูปของโทเค็น PCB) ไปจัดเรียงลำดับลงในคิวจัดการข้อมูล Scheduling Queue เพื่อรอรับการจัดสรรทรัพยากรตามลำดับขั้น โดยแบ่งออกเป็น 3 คิวแกนหลักในระบบ:

    Job Queue

    คิวรวบรวมรายชื่อโพรเซสทั้งหมดที่เกิดขึ้นภายในระบบคอมพิวเตอร์ ณ ช่วงเวลานั้นๆ ไม่ว่าจะสแตนด์บายอยู่บนดิสก์หรือแรม

    Ready Queue

    คิวเก็บรายชื่อโพรเซสทั้งหมดที่โหลดเข้ามาฝังตัวอยู่ภายในหน่วยความจำหลัก (RAM) เรียบร้อยแล้ว มีสภาพพร้อมคำนวณรันคำสั่ง ขาดเพียงรอตัวจัดสรรส่งเข้าสู่ CPU

    Device Queue

    หรือ I/O Queue คือคิวของกลุ่มโพรเซสที่ติดขัดสถานะ ต้องหยุดรอความพร้อมประมวลผลข้อมูลจากอุปกรณ์อินพุตเอาต์พุตภายนอกเฉพาะตัวชิ้นนั้นๆ

    4. Flowchart แสดงวงจรชีวิตและสถานะของ Process (5-State Model)

    ในระหว่างวงรอบชีวิต โพรเซสจะถูกปรับเปลี่ยนสถานะตามแผนผังกลไก 5-State Model ซึ่งเป็นมาตรฐานโครงสร้างระบบปฏิบัติการระดับสากล โดยมีรายละเอียดขั้นตอนความเชื่อมโยงตาม Flowchart ด้านล่างนี้:

    Flowchart: โครงสร้างการย้ายสถานะโพรเซสสากล
    [ New ]
    [ Ready ]
    Dispatch
    Interrupt
    [ Running ]
    [ Terminated ]
    [ Waiting / Blocked ]
    🔄 คำอธิบายเส้นทางคิว:
    1. เมื่ออยู่สถานะ Running แล้วเกิดการเรียกใช้ I/O โพรเซสจะถูกดิ่งลงมาที่คิว Waiting
    2. เมื่ออุปกรณ์ประมวลผลข้อมูล I/O เสร็จสิ้น โพรเซสจะดีดกลับไปต่อแถวคิวที่สถานะ Ready เสมอ ไม่สามารถแซงคิวเข้าหา CPU โดยตรงได้

    5. Animation แอนิเมชันจำลองการเปลี่ยนสถานะและกลไก Context Switching

    Context Switching คือกระบวนการที่เกิดขึ้นเมื่อระบบปฏิบัติการทำการสลับเปลี่ยนงานให้ซีพียู โดย Kernel จะหยุดทำงานโพรเซสเดิม บันทึกค่ารีจิสเตอร์และสภาพแวดล้อมลง PCB เก่า และทำการย้ายไปดึงเอาค่าจาก PCB ตัวถัดไปมาสวมรอยรันต่อบนฮาร์ดแวร์ซีพียู

    นักศึกษาสามารถคลิกปุ่มรันตัวจำลองเพื่อสังเกตการไหลของโทเค็นโพรเซสและไฟสัญญาณนำทาง Context Switching แบบเรียลไทม์:

    6. วิกฤตสภาวะการติดหล่มถาวร (Deadlock)

    หากระบบปฏิบัติการจัดการคิวและทรัพยากรผิดพลาด อาจก่อให้เกิดสภาวะ Deadlock (การติดล็อกตาย) หมายถึงสถานการณ์ที่โพรเซสตั้งแต่ 2 ตัวขึ้นไป ติดคิวรอคอยทรัพยากรซึ่งกันและกันเป็นวงกลมไม่รู้จบ โดยโพรเซส A ถือครองทรัพยากรที่ 1 และรอทรัพยากรที่ 2 ส่วนโพรเซส B ถือครองทรัพยากรที่ 2 และร้องขอทรัพยากรที่ 1 ต่างฝ่ายต่างไม่สามารถขยับรันข้อคำสั่งต่อได้ ส่งผลให้โปรแกรมระบบค้างนิ่ง (System Freeze) ถาวร
