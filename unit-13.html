    Unit 13: Multi OS (Part 2)

    หน่วยที่ 13 Multi OS ตอนที่ 2

    1. โครงสร้างและการทำงานของ Boot Loader

    Boot Loader คือโปรแกรมขนาดเล็กที่จัดเก็บอยู่ในพื้นที่เฉพาะของอุปกรณ์จัดเก็บข้อมูล (เช่น Master Boot Record หรือ MBR และ EFI System Partition หรือ ESP) ทำหน้าที่โหลดเคอร์เนล (Kernel) ของระบบปฏิบัติการเข้าสู่หน่วยความจำ RAM ในระหว่างกระบวนการเปิดเครื่อง (Bootstrapping Process) สำหรับระบบปฏิบัติการแบบ Multi OS Boot Loader ถือเป็นตัวกลางสำคัญในการควบคุมและนำทางให้ผู้ใช้เลือก Boot ระบบปฏิบัติการที่ต้องการใช้งาน

    ในสถาปัตยกรรมคอมพิวเตอร์ยุคใหม่ Boot Loader จะแบ่งออกเป็น 2 มาตรฐานหลัก คือ Legacy BIOS (ใช้ MBR ในการเก็บสคริปต์สแกนพาร์ทิชัน) และ UEFI (Unified Extensible Firmware Interface) ซึ่งทำงานร่วมกับตารางพาร์ทิชันแบบ GPT (GUID Partition Table) โดย UEFI มีความยืดหยุ่นกว่าเนื่องจากสามารถโหลดตัวจัดการบูต (.efi) หลายตัวแยกกันตามแต่ละระบบปฏิบัติการได้โดยตรง

    2. GRUB (Grand Unified Bootloader)

    GRUB (ปัจจุบันใช้ GRUB 2) เป็น Boot Loader มาตรฐานยอดนิยมในฝั่งระบบปฏิบัติการ Linux ที่มีความสามารถในการจัดการ Multi OS สูงที่สุด สามารถตรวจจับและโหลดเคอร์เนลได้ทั้ง Linux, Windows, macOS และ BSD โดย GRUB อ่านค่าคอนฟิกูเรชันหลักจากไฟล์ /boot/grub/grub.cfg ซึ่งสร้างขึ้นอัตโนมัติผ่านคำสั่ง update-grub หรือ Script ใน /etc/grub.d/ และตั้งค่าทั่วไปผ่าน /etc/default/grub

    โครงสร้างและพารามิเตอร์ที่สำคัญของ GRUB ประกอบด้วย:

    • GRUB_DEFAULT: กำหนดระบบปฏิบัติการเริ่มต้นที่จะบูต เช่น 0 (ลำดับแรก) หรือ saved
    • GRUB_TIMEOUT: ระยะเวลาถอยหลัง (วินาที) ในการแสดงเมนูให้ผู้ใช้เลือก OS ก่อนที่จะเข้าสู่ OS ค่าเริ่มต้น
    • os-prober: ยูทิลิตีสแกนพาร์ทิชันอื่นในเครื่องเพื่อหา OS อื่น (เช่น Windows Boot Manager) แล้วนำมาเพิ่มในเมนู GRUB โดยอัตโนมัติ

    3. ระบบแฟ้มข้อมูล และการแบ่งปันข้อมูล (Data Sharing, NTFS, Ext4)

    ปัญหาสำคัญในการใช้งานระบบ Multi OS (เช่น Windows และ Linux ในเครื่องเดียวกัน) คือความแตกต่างของระบบแฟ้มข้อมูล (File System) ทำให้การแชร์ข้อมูลข้ามระบบปฏิบัติการต้องทำความเข้าใจกลไกและรูปแบบพาร์ทิชันดังนี้:

    • NTFS (New Technology File System): ระบบแฟ้มข้อมูลมาตรฐานของ Windows รองรับสิทธิ์การเข้าถึง (ACL) และไฟล์ขนาดใหญ่ Windows อ่าน/เขียนได้สมบูรณ์ ในขณะที่ Linux สามารถอ่าน/เขียน NTFS ได้อย่างสมบูรณ์เช่นกันผ่านเคอร์เนลไดรเวอร์ NTFS3 หรือ NTFS-3G
    • Ext4 (Fourth Extended Filesystem): ระบบแฟ้มข้อมูลมาตรฐานของ Linux มีประสิทธิภาพสูง มีระบบ Journaling ป้องกันไฟล์เสียหาย แต่ Windows ไม่รองรับการอ่าน/เขียน Ext4 โดยธรรมชาติ (ต้องใช้เครื่องมือเสริม เช่น WSL2 หรือ Ext2Fsd)
    • เทคนิค Data Sharing Partition: วิธีที่ดีที่สุดสำหรับการแชร์ไฟล์ในเครื่อง Multi OS คือการสร้างพาร์ทิชันกลางด้วยระบบแฟ้มข้อมูลที่รองรับทั้งสองฝั่ง เช่น NTFS หรือ exFAT สำหรับเก็บข้อมูลเอกสารและมัลติมีเดียโดยเฉพาะ

    4. การเมานต์พาร์ทิชันและการปรับแต่ง fstab (Mount & /etc/fstab)

    ในระบบ Linux การเข้าถึงพาร์ทิชันหรือไดรฟ์ต่าง ๆ จะต้องผ่านกระบวนการ **Mount (การเมานต์)** เพื่อเชื่อมโยงพาร์ทิชันเข้ากับไดเรกทอรีในโครงสร้างระบบไฟล์ (Tree Structure) เช่น /mnt/data หรือ /media/user/share

    เพื่อให้ระบบ Linux ทำการเมานต์พาร์ทิชันที่ต้องการให้อัตโนมัติเมื่อเปิดเครื่อง จะต้องกำหนดค่าในไฟล์ /etc/fstab (File System Table) โดยระบุรูปแบบข้อมูลดังนี้:

    UUID=XXXX-XXXX-XXXX-XXXX /mnt/data ntfs-3g defaults,uid=1000,gid=1000 0 2
    • Device Identifier: ระบุด้วย UUID (Universally Unique Identifier) ของพาร์ทิชัน
    • Mount Point: เส้นทางไดเรกทอรีปลายทางที่ต้องการเมานต์
    • File System Type: ชนิดของระบบไฟล์ เช่น ext4, ntfs-3g, vfat
    • Mount Options: ตัวเลือกการสิทธิ์ เช่น defaults, ro (Read-Only), rw (Read-Write)

    5. Recovery Mode และ Boot Repair

    เมื่อระบบ Multi OS เกิดปัญหา เช่น เมนู GRUB หายไปหลังจากอัปเดต Windows หรือไฟล์ระบบเสียหาย การกู้คืนระบบสามารถทำได้ผ่านเครื่องมือ Recovery Mode และ Boot Repair:

    • Recovery Mode: โหมดการบูตพิเศษของ Linux ที่เข้าสู่สิทธิ์ Root Shell โดยไม่ต้องใช้ GUI ช่วยให้ผู้ใช้แก้ไขไฟล์คอนฟิก ติดตั้งไดรเวอร์ หรือสั่งซ่อมแซมพาร์ทิชันด้วยคำสั่ง fsck ได้
    • Boot Repair: เครื่องมือ GUI และ Command-line ยอดนิยมบน Linux สำหรับซ่อมแซม GRUB และโครงสร้าง EFI โดยอัตโนมัติ แก้ไขอาการ GRUB Boot Error, MBR Overwritten หรือ Windows อัปเดตทับพื้นที่บูต

    6. การแก้ปัญหา Boot ไม่ขึ้น และตัวอย่างสถานการณ์จริง

    สถานการณ์ปัญหาที่พบบ่อยที่สุดในระบบ Dual Boot (Windows + Linux) และแนวทางการแก้ไข:

    1. สถานการณ์ที่ 1: ติดตั้ง Windows หลัง Linux แล้ว GRUB หาย บูตเข้าได้แต่ Windows
      สาเหตุ: ตัวติดตั้ง Windows จะเขียนทับ Master Boot Record (MBR) หรือเปลี่ยน Boot Order ใน NVRAM ของ UEFI ให้ชี้ไปที่ Windows Boot Manager
      แนวทางแก้ไข: บูตด้วย Linux Live USB แล้วใช้คำสั่ง chroot เข้าสู่ระบบเดิม จากนั้นสั่งรัน grub-install /dev/sda และ update-grub หรือใช้เครื่องมือ Boot Repair สั่ง Recommended Repair
    2. สถานการณ์ที่ 2: บูตเข้า Linux แล้วไม่สามารถเมานต์หรือเขียนข้อมูลลงไดรฟ์ NTFS ของ Windows ได้ (Operation not permitted)
      สาเหตุ: ฟีเจอร์ Fast Startup ใน Windows ปิดเครื่องไม่สมบูรณ์ (เข้าสู่สถานะ Hibernation) ทำให้ไฟล์ระบบ NTFS ล็อกไว้
      แนวทางแก้ไข: บูตเข้า Windows แล้วไปที่ Power Options -> ปิดการใช้งาน Fast Startup หรือสั่ง Shutdown แบบ Full Shutdown (กด Shift ค้างขณะคลิก Shut down)
    3. สถานการณ์ที่ 3: เปิดเครื่องแล้วขึ้นหน้าจอ grub rescue>
      สาเหตุ: GRUB หาไฟล์คอนฟิกูเรชันใน /boot/grub ไม่พบ อาจเกิดจากการลบหรือย้ายพาร์ทิชัน Linux
      แนวทางแก้ไข: ใช้คำสั่ง ls ค้นหาพาร์ทิชันเดิมของ Linux กำหนดค่า set root=(hd0,msdosX) และ set prefix=(hd0,msdosX)/boot/grub แล้วสั่ง insmod normal ตามด้วย normal เพื่อบูตเข้าชั่วคราวแล้วซ่อมแซม GRUB

    7. ผังกระบวนการกู้คืน Boot Loader (Boot Recovery Mapping)

    แผนภาพเปรียบเทียบขั้นตอนการทำงานเมื่อเกิดปัญหา Boot ไม่ขึ้นในระบบ Multi OS และเส้นทางการกู้คืนผ่าน Live Media / Boot Repair:

    กระบวนการกู้คืน GRUB Boot Loader (Boot Repair Flow)
    1. เกิดปัญหา Boot Failure / MBR Overwritten
    เกิดจาก Windows Update ทับพื้นที่การบูต หรือพาร์ทิชัน GRUB เสียหาย
    2. บูตผ่าน Live USB & Boot Repair / Chroot
    ใช้ Live Media เข้าไปติดตั้ง GRUB ใหม่ลง ESP/MBR และรัน update-grub

    8. Animation จำลองกระบวนการกู้คืน GRUB Boot Loader

    แบบจำลองการแก้ไขสถานการณ์เมื่อ Windows เขียนทับพื้นที่บูต (MBR/NVRAM) ส่งผลให้ GRUB หายไป และการใช้ Live USB เพื่อซ่อมแซมและกู้คืนเมนู Multi Boot กลับคืนมา

    9. ตัวอย่างตารางการตั้งค่า fstab สำหรับการเมานต์ Multi OS

    พิจารณาสถานการณ์การกำหนดค่าไดรฟ์และพาร์ทิชันต่าง ๆ ในไฟล์ /etc/fstab เพื่อให้ระบบ Linux เมานต์พาร์ทิชันอัตโนมัติเมื่อเปิดเครื่อง:

    ตัวอย่างโครงสร้างไฟล์ /etc/fstab ในระบบ Dual Boot:

    ตัวอย่างการกำหนดสิทธิ์เมานต์พาร์ทิชัน Linux (Ext4), Windows (NTFS) และพาร์ทิชันแชร์ข้อมูล:

    • # System Root (Ext4): UUID=a1b2c3d4-1234... / ext4 errors=remount-ro 0 1 (พาร์ทิชันระบบ Linux)
    • # Data Share Partition (NTFS): UUID=5678-EF90 /mnt/SharedData ntfs-3g defaults,uid=1000,gid=1000 0 0 (แชร์ข้อมูล Win/Linux)
    • # Windows C: Drive (Read-Only): UUID=1234-ABCD /mnt/WindowsC ntfs-3g ro,defaults 0 0 (อ่านอย่างเดียว ป้องกันไฟล์ระบบเสียหาย)
