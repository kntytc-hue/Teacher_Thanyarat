Unit 02: Functions and Operations of Operating Systems

หน่วยที่ 2 หน้าที่และหลักการทำงานของระบบปฏิบัติการ

วัตถุประสงค์การเรียนรู้
  • อธิบายความสัมพันธ์และหน้าที่ของ Kernel และ Shell ได้อย่างชัดเจน
  • จำแนกความแตกต่างระหว่างระบบแบบอินเทอร์เฟซ CLI และ GUI ได้
  • วิเคราะห์กลไกการสลับและประมวลผลทรัพยากรของระบบปฏิบัติการแต่ละประเภทได้ถูกต้อง
สมรรถนะรายวิชาที่พึงมี
  • แสดงความรู้เกี่ยวกับบทบาทของระบบปฏิบัติการในการบริหารจัดการทรัพยากรฮาร์ดแวร์หลัก
  • จัดจำแนกประเภทของระบบปฏิบัติการตามเงื่อนไขความต้องการของหน้างานเทคโนโลยีเครือข่าย

1. สถาปัตยกรรมการปฏิสัมพันธ์ของระบบ (Kernel, Shell, และ User Interface)

การทำงานภายในของระบบปฏิบัติการไม่ได้เชื่อมต่อกับผู้ใช้ในชั้นเดียว แต่แบ่งแยกสิทธิเป็นเลเยอร์เพื่อเสถียรภาพและความปลอดภัย โดยโครงสร้างหลักจะประกอบด้วย Kernel (เคอร์เนล) ซึ่งเป็นแกนกลางชั้นในสุดที่ทำงานอยู่ในสถานะสิทธิ์สูงสุด (Privileged Mode/Kernel Space) มีหน้าที่ติดต่อ สั่งการ และจัดสรรพลังงานไฟฟ้าให้ฮาร์ดแวร์คอมพิวเตอร์โดยตรง ทั้งควบคุมรอบเวลาสัญญาณนาฬิกาของ CPU และจัดสรรหน้าหน่วยความจำหลัก

ถัดมาในเลเยอร์ที่ห่อหุ้มเคอร์เนลไว้คือ Shell (เชลล์) ทำหน้าที่เป็นตัวแปลชุดคำสั่ง (Command Interpreter) คอยรับสัญญาณความต้องการจากโปรแกรมภายนอกหรือผู้ใช้ ตรวจสอบความถูกต้องทางไวยากรณ์โครงสร้าง แล้วส่งผ่านสัญญาณในรูปแบบ System Calls ไปให้ระดับ Kernel ประมวลผลต่อ โดยผู้ใช้จะส่งผ่านความต้องการผ่านตัวกลางที่เรียกว่า User Interface (UI) ซึ่งแยกเป็น 2 โครงสร้างย่อย:

  • CLI (Command Line Interface): การสั่งการผ่านบรรทัดข้อความตัวอักษรล้วน เช่น ระบบทุบคำสั่งทราฟฟิกคอนโซล Bash ใน Linux หรือ CMD ใน Windows ประหยัดหน่วยความจำ RAM สูงสุดและทำงานได้รวดเร็วเด็ดขาด
  • GUI (Graphical User Interface): การสั่งการผ่านสัญลักษณ์ภาพกราฟิก (WIMP: Windows, Icons, Menus, Pointer) ควบคุมผ่านเมาส์หรือการทัชสกรีนหน้าจอ ช่วยลดความยุ่งยากในการจำคำสั่งของผู้เรียนและผู้ใช้งานทั่วไป

2. หน้าที่หลักในการจัดการทรัพยากร (Resource Management)

ระบบปฏิบัติการจะทำหน้าที่เสมือนผู้จัดสรรทรัพยากร (Resource Allocator) ของคอมพิวเตอร์ที่มีจำกัด โดยโฟกัสหน้าที่หลักใน 4 แกนสำคัญ:

  1. การจัดการโพรเซส (Process Management): ควบคุม จัดคิว วนเวลา และสลับโปรแกรมงานเข้าประมวลผลใน CPU รวมถึงแก้ไขสภาวะติดขัดอย่าง Deadlock
  2. การจัดการหน่วยความจำหลัก (Memory Management): ทำหน้าที่ Allocate จองพื้นที่ RAM ให้แอปพลิเคชัน และทำหน้าที่คายคืนพื้นที่ (Deallocate) ทันทีเมื่อแอปนั้นปิดตัว เพื่อป้องกันปัญหาความจำรั่ว (Memory Leak)
  3. การจัดการระบบไฟล์ (File System Management): จัดเตรียมสารบบและการเข้าถึงตรรกะแฟ้มข้อมูลในฮาร์ดดิสก์/SSD
  4. การจัดการอุปกรณ์รับส่งข้อมูล (I/O Management): ประสานงานสัญญาณรับเข้าและแสดงผลผ่านซอฟต์แวร์ไดรเวอร์เฉพาะทาง

3. สถาปัตยกรรมและประเภทของระบบปฏิบัติการ

ระบบปฏิบัติการในปัจจุบันสามารถจำแนกออกตามโครงสร้างการแบ่งเวลาและการคำนวณสวิตช์สัญญาณได้ดังนี้:

  • Single Tasking & Multi Tasking: งานเดี่ยวรันทีละคิว (เช่น MS-DOS) เทียบกับระบบงานหลากหน้าต่างสลับประมวลผลพร้อมกัน (เช่น Windows, macOS)
  • Multi Programming & Multi Processing: มัลติโปรแกรมมิ่งคือการโหลดงานหลายตัวพักไว้ในแรมเพื่อเตรียมสลับรันเมื่อตัวแรกรอข้อมูล I/O ส่วนมัลติโพรเซสซิ่งคือการรันงานกระจายบนชิปตัวประมวลผลซีพียูจริง (Cores) หลายตัวพร้อมกันทางกายภาพ
  • Distributed OS & Network OS: ระบบแบบกระจายคำนวณเชื่อมต่ออิสระหลายเครื่องแยกเครื่องกันแต่จำลองเสมือนรันบนเครื่องเดียว เทียบกับระบบเครือข่ายเซิร์ฟเวอร์ที่เน้นความปลอดภัยเรื่องสิทธิ์ (เช่น Windows Server, Red Hat)
  • Embedded OS & Real-Time OS (RTOS): โอเอสแบบฝังขนาดกะทัดรัดบนอุปกรณ์ไฟฟ้า เทียบกับระบบปฏิบัติการเวลาจริงที่เน้นความเที่ยงตรงด้านการประมวลผลแบบห้ามเรทเดดไลน์วินาทีเด็ดขาด (เช่น กล่องควบคุมเบรกรถยนต์ ABS หรือระบบขีปนาวุธ)

4. ตารางเปรียบเทียบประเภทของระบบปฏิบัติการ พร้อมตัวอย่างจริง

ประเภทระบบปฏิบัติการ กลไกการทำงานหลัก ข้อดี / คุณลักษณะเด่น ตัวอย่างซอฟต์แวร์จริง
Single Tasking รันโพรเซสเดียวจนจบจึงรับงานใหม่ สถาปัตยกรรมง่าย ไม่แย่งทรัพยากร MS-DOS, FreeDOS
Multi Tasking สลับเวลาซีพียูแบ่งส่วนให้แอปต่างๆ ผู้ใช้งานทำงานได้ต่อเนื่อง ยืดหยุ่นสูง Windows 11, macOS, Linux
Multi Processing แบ่งเธรดงานกระจายลงซีพียูหลาย Core จริง คำนวณงานใหญ่เสร็จเร็ว ทนทานสูง Windows Server, Linux Core มัลติคอร์
Real-Time OS (RTOS) ตอบสนองทันทีภายใต้เงื่อนไขเวลาจำกัด รับประกันเวลาประมวลผลเคร่งครัด VxWorks, FreeRTOS, QNX
