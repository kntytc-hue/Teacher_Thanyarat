    Unit 11: Multi OS Preparation

    หน่วยที่ 11 การเตรียมติดตั้งหลายระบบ

    1. การเตรียมการติดตั้งหลายระบบปฏิบัติการ (Multi OS Preparation)

    การติดตั้งหลายระบบปฏิบัติการในเครื่องเดียวกัน (Multi-boot / Dual Boot) คือกระบวนการจัดสรรทรัพยากรคอมพิวเตอร์ให้สามารถรองรับระบบปฏิบัติการมากกว่าหนึ่งระบบ เช่น Windows ร่วมกับ Linux (Ubuntu) หรือ Windows หลายเวอร์ชัน การเตรียมความพร้อมอย่างถูกต้องถือเป็นขั้นตอนสำคัญสูงสุด เนื่องจากหากไม่มีการวางแผนที่ดี อาจส่งผลให้ข้อมูลสูญหาย ตัวบูตระบบเสียหาย หรือไม่สามารถเข้าใช้งานระบบปฏิบัติการเดิมได้

    ปัจจัยหลักในการเตรียมความพร้อมครอบคลุมการตรวจสอบสถาปัตยกรรมเฟิร์มแวร์ (BIOS/UEFI) รูปแบบโครงสร้างการจัดเก็บข้อมูลดิสก์ (MBR/GPT) การแบ่งพาร์ติชัน (Partitioning) การจัดการตัวบูตระบบ (Boot Manager) ตลอดจนการสำรองข้อมูลและการวางแผนพื้นที่จัดเก็บดิสก์อย่างเป็นระบบ

    2. โครงสร้างตารางพาร์ติชัน: MBR และ GPT

    ตารางพาร์ติชัน (Partition Table) เป็นโครงสร้างข้อมูลที่จัดเก็บรายละเอียดเกี่ยวกับการแบ่งส่วนพื้นที่จัดเก็บข้อมูลบนฮาร์ดดิสก์หรือ SSD มี 2 รูปแบบหลัก ได้แก่:

    • Master Boot Record (MBR): เป็นมาตรฐานดั้งเดิมที่ใช้กับระบบ BIOS พัฒนาขึ้นตั้งแต่ยุค IBM PC รองรับความจุฮาร์ดดิสก์สูงสุดไม่เกิน 2 Terabytes (TB) และจำกัดการสร้าง Primary Partition ได้ไม่เกิน 4 พาร์ติชัน ข้อมูล MBR ถูกเก็บไว้ที่ Sector แรกของดิสก์ หาก Sector นี้เสียหาย ระบบจะไม่สามารถบูตได้ทันที
    • GUID Partition Table (GPT): เป็นมาตรฐานใหม่ที่พัฒนาขึ้นมาพร้อมกับ UEFI รองรับความจุฮาร์ดดิสก์ได้สูงถึง 9.4 Zettabytes (ZB) สามารถสร้าง Primary Partition ได้สูงสุดถึง 128 พาร์ติชัน (บน Windows) GPT มีการเก็บสำรองตารางพาร์ติชันไว้ที่ส่วนท้ายของดิสก์ (Backup Header) และใช้ระบบ CRC32 ในการตรวจสอบความสมบูรณ์ของข้อมูล

    3. ประเภทของพาร์ติชัน (Partition Types)

    การจัดแบ่งพื้นที่ดิสก์โดยเฉพาะในระบบ MBR จำเป็นต้องเข้าใจประเภทของพาร์ติชัน เพื่อก้าวข้ามข้อจำกัดการแบ่งพื้นที่ ได้แก่:

    1. Primary Partition (พาร์ติชันหลัก): พาร์ติชันที่ใช้สำหรับการติดตั้งระบบปฏิบัติการและไฟล์บูตหลัก ระบบ MBR สามารถมี Primary Partition ได้สูงสุด 4 พาร์ติชันต่อหนึ่งดิสก์
    2. Extended Partition (พาร์ติชันส่วนขยาย): พาร์ติชันพิเศษที่สร้างขึ้นมาทดแทน Primary Partition ได้ 1 ตำแหน่ง เพื่อข้ามข้อจำกัด 4 พาร์ติชันของ MBR โดย Extended Partition จะไม่เก็บข้อมูลโดยตรง แต่ทำหน้าที่เป็นคอนเทนเนอร์ครอบ Logical Partition ไว้ภายใน
    3. Logical Partition (พาร์ติชันตรรกะ): พาร์ติชันที่ถูกสร้างขึ้นอยู่ภายใน Extended Partition สามารถสร้างได้หลายพาร์ติชัน เหมาะสำหรับจัดเก็บข้อมูลทั่วไป หรือติดตั้งระบบปฏิบัติการรอง เช่น Linux

    4. สถาปัตยกรรมเฟิร์มแวร์: BIOS และ UEFI

    เฟิร์มแวร์บนเมนบอร์ดทำหน้าที่ตรวจสอบอุปกรณ์ฮาร์ดแวร์ (POST) และส่งต่อการทำงานให้ตัวบูตระบบ มี 2 สถาปัตยกรรมหลัก:

    • Legacy BIOS (Basic Input/Output System): ระบบอินเทอร์เฟซดั้งเดิม ประมวลผลแบบ 16-bit รองรับดิสก์แบบ MBR เท่านั้น ทำงานโดยอ่านรหัสบูตจาก Master Boot Record ที่ Sector แรก
    • UEFI (Unified Extensible Firmware Interface): สถาปัตยกรรมสมัยใหม่ ทำงานแบบ 32-bit หรือ 64-bit รองรับกราฟิกอินเทอร์เฟซ การทำงานร่วมกับ GPT บูตรวดเร็ว มีไดรเวอร์ในตัว และสามารถอ่านระบบไฟล์ FAT32 ได้โดยตรงโดยไม่ต้องพึ่งพารหัส MBR

    5. ตัวจัดการการบูตและฟีเจอร์ความปลอดภัย (Boot Manager & Secure Boot)

    การรันหลายระบบปฏิบัติการจำต้องอาศัยกลไกการสลับและควบคุมความปลอดภัยในระดับบูตโหลดเดอร์:

    • Boot Manager (ตัวจัดการการบูต): โปรแกรมระดับฐานรากที่ทำหน้าที่แสดงเมนูให้ผู้ใช้เลือกระบบปฏิบัติการที่ต้องการรันเมื่อเปิดเครื่อง เช่น Windows Boot Manager (BCD) สำหรับ Windows หรือ GRUB (Grand Unified Bootloader) สำหรับ Linux โดย GRUB สามารถตรวจพบและเรียกบูต Windows ได้อย่างมีประสิทธิภาพ
    • Secure Boot: ฟีเจอร์ความปลอดภัยในมาตรฐาน UEFI ทำหน้าที่ตรวจสอบลายเซ็นดิจิทัล (Digital Signature) ของไดรเวอร์และตัวบูตระบบ เพื่อป้องกัน Malicious Code หรือ Rootkit แอบแฝงเข้ามาทำงานก่อนระบบปฏิบัติการจะโหลด อย่างไรก็ตาม ในการติดตั้ง Linux บางดิสโทร อาจจำเป็นต้องตั้งค่า Key หรือปิด Secure Boot ชั่วคราวเพื่อให้ติดตั้งสำเร็จ

    6. การสำรองข้อมูลและการวางแผนพื้นที่ Disk

    ก่อนดำเนินการปรับแต่งโครงสร้างพาร์ติชันและติดตั้ง Multi-OS จำเป็นต้องดำเนินการเตรียมการสำคัญ 2 ประการ:

    • การสำรองข้อมูลก่อนติดตั้ง (Pre-installation Data Backup): ทำการสำรองข้อมูลสำคัญ (Full Backup หรือ Selective Backup) ลงในสื่อจัดเก็บภายนอก (External Hard Drive, Cloud Storage) รวมทั้งสร้าง System Restore Point หรือ Disk Image เพื่อรับมือกับกรณีเกิดความผิดพลาด MBR/GPT สดุดเสียหาย หรือการฟอร์แมตผิดพาร์ติชัน
    • การวางแผนพื้นที่ Disk (Disk Space Allocation Planning): คำนวณและจัดสรรขนาดพาร์ติชันให้เหมาะสมตามความต้องการของแต่ละ OS เช่น Windows ต้องใช้พื้นที่อย่างน้อย 64 GB ขึ้นไปสำหรับ OS และโปรแกรมประยุกต์ ส่วน Linux ต้องจัดสรรพื้นที่สำหรับ Root Partition (/), Swap Space (หน่วยความจำเสมือนบนดิสก์) และ /home พาร์ติชันสำหรับการเก็บข้อมูลผู้ใช้

    7. ผังกระบวนการการทำงานและเปรียบเทียบสถาปัตยกรรม Multi-OS

    เปรียบเทียบโครงสร้างการเชื่อมโยงการทำงานระหว่าง Legacy BIOS + MBR กับ UEFI + GPT ในการติดตั้งหลายระบบปฏิบัติการ:

    เปรียบเทียบเส้นทางการบูต Multi-OS Architecture
    Legacy Architecture (BIOS + MBR)
    • เฟิร์มแวร์: Legacy BIOS
    • โครงสร้างดิสก์: MBR (สูงสุด 4 Primary)
    • ตัวสลับบูต: GRUB เขียนทับ MBR Sector 0
    • ความปลอดภัย: ไม่มี Secure Boot
    Modern Architecture (UEFI + GPT)
    • เฟิร์มแวร์: UEFI (32/64-bit Execution)
    • โครงสร้างดิสก์: GPT (รองรับ 128 Primary)
    • ตัวสลับบูต: EFI System Partition (ESP)
    • ความปลอดภัย: รองรับ Secure Boot

    9. ตัวอย่างตารางเปรียบเทียบการจัดสรรพื้นที่พาร์ติชัน (Disk Partition Planning)

    พิจารณาสถานการณ์การวางแผนจัดสรรดิสก์ขนาด 500 GB สำหรับการทำ Dual Boot ระหว่าง Windows 11 และ Ubuntu Linux:

    ตารางการจัดสรรดิสก์ (Disk Allocation Table Example):
    พาร์ติชัน ระบบไฟล์ (File System) ขนาดแนะนำ จุดประสงค์การใช้งาน
    EFI System Partition (ESP) FAT32 100 - 512 MB เก็บไฟล์ Bootloader (Windows Boot Manager / GRUB)
    Windows (C:) NTFS 200 GB ติดตั้งระบบปฏิบัติการ Windows 11 และโปรแกรมหลัก
    Linux Root (/) ext4 80 GB ติดตั้งระบบปฏิบัติการ Linux และแพ็กเกจซอฟต์แวร์
    Linux Swap swap 8 - 16 GB พื้นที่จำลองหน่วยความจำเสมือนสำหรับ Linux
    Data Drive (Shared) NTFS / exFAT ~200 GB พื้นที่จัดเก็บข้อมูลตรงกลางที่อ่าน-เขียนได้ทั้ง Windows และ Linux
