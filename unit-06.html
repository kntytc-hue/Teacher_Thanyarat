    Unit 06: Virtual Memory Management

    หน่วยที่ 6 หน่วยความจำเสมือน

    1. แนวคิดหน่วยความจำเสมือน (Virtual Memory Concept)

    หน่วยความจำเสมือน (Virtual Memory) เป็นเทคนิคการจัดการหน่วยความจำที่แยกส่วนของหน่วยความจำเชิงตรรกะของผู้ใช้ (Logical Memory) ออกจากหน่วยความจำทางกายภาพ (Physical Memory หรือ RAM) อย่างเด็ดขาด ช่วยให้โปรแกรมเมอร์สามารถเขียนโปรแกรมที่มีขนาดใหญ่กว่าเนื้อที่ของ RAM จริงที่มีอยู่ในระบบคอมพิวเตอร์ได้

    หัวใจหลักของหน่วยความจำเสมือนคือ การนำพื้นที่บางส่วนของหน่วยความจำสำรอง เช่น ฮาร์ดดิสก์ (Hard Disk) หรือโซลิดสเตตไดรฟ์ (SSD) มาทำหน้าที่จำลองเป็นหน่วยความจำหลักชั่วคราว ทำให้ระบบปฏิบัติการสามารถรันโปรเซสได้พร้อมกันมากขึ้น และใช้ทรัพยากร RAM ที่มีอยู่อย่างจำกัดได้อย่างคุ้มค่าสูงสุด

    2. ความต้องการหน้าเมื่อร้องขอ (Demand Paging)

    Demand Paging เป็นระบบที่นำมาใช้บ่อยที่สุดในการติดตั้งหน่วยความจำเสมือน โดยมีหลักการคือ **"จะโหลดหน้า (Page) ของโปรแกรมเข้าสู่ RAM ก็ต่อเมื่อมีความต้องการใช้งานหน้านั้นจริงๆ เท่านั้น"** ในตอนเริ่มต้นรันโปรเซส หน้าทั้งหมดจะอยู่บนดิสก์ และจะค่อยๆ ถูกดึงเข้ามาใน RAM เมื่อถูกเรียกใช้งาน

    กลไกนี้อาศัยบิตตรวจสอบความถูกต้อง (Valid-Invalid Bit) ในตารางหน้า (Page Table) เพื่อระบุสถานะ:

    • Valid (V): หน้านั้นถูกโหลดเข้ามาและสถิตอยู่ใน RAM เรียบร้อยแล้ว CPU เข้าถึงได้ทันที
    • Invalid (I): หน้านั้นยังไม่ได้ถูกโหลดเข้ามาใน RAM (ยังอยู่บนดิสก์) หรือเป็นตำแหน่งที่ไม่มีอยู่จริง

    3. การเกิดความผิดพลาดของหน้า (Page Fault)

    เมื่อ CPU พยายามเข้าถึงตำแหน่งที่อยู่ใดๆ แล้วพบว่าบิตใน Page Table ของหน้านั้นเป็น **Invalid** จะทำให้เกิดเหตุการณ์ขัดจังหวะที่เรียกว่า **Page Fault (ความผิดพลาดของหน้า)** ซึ่งระบบปฏิบัติการจะเข้ามาจัดการตามขั้นตอนดังนี้:

    1. ระบบปฏิบัติการจะหยุดการทำงานของโปรเซสนั้นชั่วคราวเพื่อตรวจสอบว่าเป็นการเข้าถึงข้อมูลที่ถูกต้องหรือไม่
    2. หากถูกต้อง ระบบจะไปค้นหาและระบุตำแหน่งของหน้านั้นบนหน่วยความจำสำรอง (ดิสก์)
    3. ค้นหาเฟรมที่ว่าง (Free Frame) ใน RAM เพื่อเตรียมนำหน้าดังกล่าวเข้ามาจัดเก็บ
    4. เริ่มกระบวนการอ่านข้อมูลจากดิสก์เข้ามายังเฟรมที่ว่างใน RAM (Disk I/O)
    5. เมื่อดาวน์โหลดเสร็จ จะอัปเดตค่าตารางหน้า (Page Table) เปลี่ยนสถานะจาก Invalid เป็น Valid
    6. คืนคำสั่งเดิมให้ CPU เพื่อเริ่มต้นประมวลผลคำสั่งที่หยุดชะงักไปใหม่อีกครั้ง

    4. การแทนที่หน้า (Page Replacement)

    ในกรณีที่เกิด Page Fault แต่พื้นที่ของ RAM เต็มหมดแล้วและไม่มีเฟรมว่างเหลืออยู่เลย ระบบปฏิบัติการจะต้องเลือกเฟรมใดเฟรมหนึ่งที่อยู่ใน RAM ออกไปเก็บไว้ในดิสก์ก่อน (เรียกว่า **Swap Out** หรือ Victim Page) เพื่อเคลียร์พื้นที่ให้เป็นเฟรมว่าง จากนั้นจึงจะนำหน้าใหม่ที่ต้องการโหลดเข้ามาแทนที่ได้ (เรียกว่า **Swap In**)

    เพื่อลดจำนวนครั้งในการเขียนข้อมูลกลับไปยังดิสก์ ระบบมักใช้ **Modify Bit (Dirty Bit)** เพื่อตรวจสอบว่าหน้านั้นเคยโดนแก้ไขข้อมูลระหว่างอยู่ใน RAM หรือไม่ หากไม่เคยถูกแก้ไข ระบบสามารถเขียนทับหน้านั้นได้ทันทีโดยไม่ต้องทำการ Write กลับลงไปในดิสก์ซ้ำ

    5. อัลกอริทึมการแทนที่หน้า (Page Replacement Algorithms)

    ประสิทธิภาพของหน่วยความจำเสมือนขึ้นอยู่กับอัลกอริทึมในการเลือกหน้าที่จะถูกเตะออก (Victim Page) เพื่อให้เกิด Page Fault น้อยที่สุด มี 3 อัลกอริทึมหลักดังนี้:

    • First-In, First-Out (FIFO): หน้าใดที่ถูกโหลดเข้ามาใน RAM เป็นอันดับแรกสุด จะถูกเลือกให้ออกจาก RAM เป็นอันดับแรกสุดเมื่อพื้นที่เต็ม แต่อาจเกิดปรากฏการณ์ Belady's Anomaly (การเพิ่มจำนวนเฟรมแต่กลับทำให้เกิด Page Fault มากขึ้น)
    • Optimal (OPT): อัลกอริทึมในอุดมคติ โดยจะเลือกเตะหน้าที่จะ **ไม่ถูกเรียกใช้งานเลยเป็นระยะเวลานานที่สุดในอนาคต** อัลกอริทึมนี้ให้ผลลัพธ์ดีที่สุดแต่ไม่สามารถนำมาใช้จริงในระบบปฏิบัติการได้ เนื่องจากระบบไม่สามารถทำนายอนาคตล่วงหน้าได้
    • Least Recently Used (LRU): เลือกเตะหน้าที่มีการ **ถูกเข้าถึงหรือเรียกใช้งานย้อนหลังยาวนานที่สุดในอดีต** (ใช้สมมติฐานว่าหน้าเก่าที่ไม่ได้ใช้นานแล้ว น่าจะไม่ถูกใช้ต่อในเร็วๆ นี้) เป็นอัลกอริทึมที่นิยมนำมาใช้งานจริงมากที่สุด

    6. สภาวะการสลับหน้าอย่างรุนแรง (Thrashing)

    สภาวะ Thrashing คือ สถานการณ์ที่โปรเซสใช้เวลาส่วนใหญ่ไปกับ **การสลับหน้าเข้าและออก (Swapping In/Out)** ระหว่าง RAM กับดิสก์ มากกว่าเวลาที่ใช้ในการประมวลผลคำสั่งจริงของ CPU ส่งผลให้ประสิทธิภาพของระบบคอมพิวเตอร์ตกลงอย่างรุนแรงและเกิดการค้าง (Freeze)

    สาเหตุหลักเกิดจากการที่ระบบรันโปรเซสพร้อมกันมากเกินไป (High Degree of Multiprogramming) จนทำให้เฟรมของ RAM ที่จัดสรรให้แต่ละโปรเซสมีไม่เพียงพอต่อการทำงานของกลุ่มคำสั่งชุดนั้นๆ (Working Set) วิธีแก้ไขคือต้องลดจำนวนโปรเซสที่รันอยู่ หรือเพิ่มขนาดของหน่วยความจำกายภาพ (RAM)

    7. ผังกระบวนการแลกเปลี่ยนข้อมูลหน่วยความจำ (Virtual to Physical Mapping)

    แผนภาพการแปลงและเชื่อมโยงข้อมูลระหว่างพื้นที่เสมือนบนดิสก์และพื้นที่จัดเก็บจริงบนแผงหน่วยความจำหลัก RAM ผ่านกลไกจัดสรร:

    กระบวนการ Swapping / Demand Paging
    Virtual Memory (บนดิสก์)
    พื้นที่จองขนาดใหญ่ที่ถูกสร้างขึ้นเพื่อขยายขอบเขตการทำงานของโปรแกรม
    Physical RAM (บนบอร์ดจริง)
    แผงวงจรความเร็วสูงที่มีขนาดจำกัด สำหรับใช้ประมวลผลงาน Active จริง

    8. Animation แอนิเมชันจำลองขั้นตอนการเกิด Page Fault และ Swap Space

    แบบจำลองสถานการณ์จำลองขณะที่ CPU มีการร้องขอหน้าข้อมูลที่เป็นโมเดลเสมือน (Virtual Page) แล้วตรวจสอบผ่าน MMU พบสถานะ Invalid จึงดึงข้อมูลสลับบล็อกจากหน่วยความจำสำรอง (Swap Space) เข้าแทนที่ RAM จริง

    นักศึกษาสามารถคลิกปุ่มรันตัวจำลองเพื่อดูการทำ Page Replacement:

    Virtual Page
    MMU (Check)
    Physical RAM
    จำลองสถานะข้อมูลบน RAM และดิสก์ (Swap Space)
    Page 1 (ใน RAM) Page 2 (ใน RAM) Page 3 (บนดิสก์) Page 4 (บนดิสก์)

    9. ตัวอย่างตารางการคำนวณแทนที่หน้า (Page Replacement Tracing)

    พิจารณาสถานการณ์จำลองการป้อนสายอ้างอิงหน้าข้อมูล (Page Reference String) เพื่อเปรียบเทียบหาจำนวนครั้งการเกิด Page Fault

    สถานการณ์จำลองสำหรับการศึกษา:

    สมมติให้ระบบคอมพิวเตอร์มีจำนวนเฟรมว่างใน RAM ทั้งหมด 3 เฟรม และมีลำดับการเรียกหน้าดังนี้:

    7, 0, 1, 2, 0, 3

    ผลลัพธ์ขั้นตอนการทำงานด้วยอัลกอริทึม FIFO:

    • เรียกหน้า 7 -> RAM ว่าง -> บรรจุ [7, -, -] (เกิด Page Fault ครั้งที่ 1)
    • เรียกหน้า 0 -> RAM ว่าง -> บรรจุ [7, 0, -] (เกิด Page Fault ครั้งที่ 2)
    • เรียกหน้า 1 -> RAM ว่าง -> บรรจุ [7, 0, 1] (เกิด Page Fault ครั้งที่ 3)
    • เรียกหน้า 2 -> RAM เต็ม -> เตะหน้า 7 ที่เก่าสุดออก -> บรรจุ [2, 0, 1] (เกิด Page Fault ครั้งที่ 4)
    • เรียกหน้า 0 -> มีหน้า 0 อยู่แล้วใน RAM -> บรรจุ [2, 0, 1] (Page Hit! ไม่เกิด Fault)
    • เรียกหน้า 3 -> RAM เต็ม -> เตะหน้า 0 ที่เก่าเป็นลำดับถัดมาออก -> บรรจุ [2, 3, 1] (เกิด Page Fault ครั้งที่ 5)

    แบบทดสอบหลังเรียนประจำหน่วยการเรียนที่ 6
