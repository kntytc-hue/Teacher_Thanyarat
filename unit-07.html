    Unit 07: Input/Output Management

    หน่วยที่ 7 การจัดการ I/O

    1. การจัดการไอโอ (I/O Management)

    หน้าที่ที่สำคัญประการหนึ่งของระบบปฏิบัติการคือการควบคุมอุปกรณ์อินพุตและเอาต์พุต (I/O Devices) ทั้งหมดของคอมพิวเตอร์ ระบบปฏิบัติการจะต้องส่งคำสั่งไปยังอุปกรณ์เหล่านี้ ตรวจจับการขัดจังหวะ (Interrupts) และจัดการกับข้อผิดพลาดที่เกิดขึ้น รวมถึงจัดเตรียมอินเทอร์เฟซที่เรียบง่ายและใช้งานง่ายระหว่างอุปกรณ์กับส่วนที่เหลือของระบบ เพื่อให้ระบบทำงานได้อย่างมีประสิทธิภาพและเป็นอิสระจากฮาร์ดแวร์เฉพาะรุ่น

    การจัดการ I/O มีความซับซ้อนเนื่องจากอุปกรณ์มีความหลากหลายสูง ทั้งในด้านความเร็ว (เช่น คีย์บอร์ดความเร็วต่ำเทียบกับ NVMe SSD ความเร็วสูง) จุดประสงค์การใช้งาน และรูปแบบการส่งผ่านข้อมูล (แบบบล็อกหรือแบบสตรีมตัวอักษร) ระบบปฏิบัติการจึงจำเป็นต้องใช้โครงสร้างแบบเลเยอร์เพื่อซ่อนความซับซ้อนเหล่านั้นไว้

    2. โปรแกรมขับเคลื่อนอุปกรณ์ (Device Driver)

    Device Driver คือโปรแกรมซอฟต์แวร์เฉพาะที่ทำหน้าที่เป็นตัวกลางในการสื่อสารระหว่างระบบปฏิบัติการและอุปกรณ์ฮาร์ดแวร์แต่ละชิ้น รหัสคำสั่งภายใน Device Driver จะถูกเขียนขึ้นโดยผู้ผลิตฮาร์ดแวร์ เพื่อทำหน้าที่แปลงคำสั่งสากลหรือคำสั่งเชิงตรรกะของระบบปฏิบัติการ (เช่น คำสั่ง Read หรือ Write) ให้กลายเป็นคำสั่งควบคุมระดับต่ำ (Low-level commands) ที่บอร์ดควบคุมอุปกรณ์ (Device Controller) เข้าใจได้

    หากไม่มี Device Driver ระบบปฏิบัติการจะต้องบรรจุโค้ดควบคุมอุปกรณ์ทุกรุ่นทุกแบรนด์ไว้ในเคอร์เนล ซึ่งเป็นไปไม่ได้ในทางปฏิบัติ การมีโครงสร้างนี้ทำให้ระบบปฏิบัติการมีความยืดหยุ่นสูง สามารถเชื่อมต่ออุปกรณ์ใหม่ๆ ได้ทันทีเพียงแค่ติดตั้งโปรแกรมขับเคลื่อนเพิ่มเติม

    3. การขัดจังหวะ (Interrupt)

    Interrupt เป็นกลไกหลักของระบบคอมพิวเตอร์ฮาร์ดแวร์ที่ใช้แจ้งเตือน CPU เมื่อมีเหตุการณ์สำคัญเกิดขึ้นในอุปกรณ์ I/O ช่วยให้ CPU ไม่ต้องเสียเวลาคอยตรวจสอบสถานะของอุปกรณ์อยู่ตลอดเวลา (Polling) ซึ่งทำให้สูญเสียรอบการประมวลผลไปโดยเปล่าประโยชน์

    เมื่ออุปกรณ์ทำงานเสร็จสิ้นหรือต้องการความช่วยเหลือ มันจะส่งสัญญาณผ่านสาย Interrupt Signal ไปยังตัวควบคุมการขัดจังหวะ จากนั้น CPU จะหยุดการทำงานของโปรเซสปัจจุบันชั่วคราว บันทึกสถานะการทำงาน (Context) ลงในสแต็ก แล้วกระโดดไปทำงานยังฟังก์ชันบริการการขัดจังหวะที่เรียกว่า Interrupt Service Routine (ISR) เพื่อจัดการข้อมูลจากอุปกรณ์นั้นๆ เมื่อจัดการเสร็จแล้วจึงจะดึงสถานะเดิมกลับมาเพื่อประมวลผลงานค้างต่อไป

    4. การเข้าถึงหน่วยความจำโดยตรง (DMA - Direct Memory Access)

    สำหรับอุปกรณ์ที่ส่งผ่านข้อมูลปริมาณมากและมีความเร็วสูง เช่น ฮาร์ดดิสก์ หรือการ์ดเครือข่ายความเร็วสูง การใช้กลไกขัดจังหวะแบบเดิม (ที่ต้องย้ายข้อมูลทีละไบต์ผ่านรีจิสเตอร์ของ CPU) จะทำให้ CPU เกิดภาระงานหนักเกินไป (CPU Overhead) ระบบคอมพิวเตอร์จึงนำกลไก DMA (Direct Memory Access) เข้ามาแก้ไขปัญหานี้

    ภายใต้การทำงานของ DMA ตัวควบคุม DMA (DMA Controller) จะเข้ามาทำหน้าที่ควบคุมบัสระบบแทน CPU เพื่อทำการคัดลอกหรือถ่ายโอนบล็อกข้อมูลระหว่างอุปกรณ์ I/O และหน่วยความจำหลัก (RAM) โดยตรง โดยกระบวนการทั้งหมดนี้ CPU จะเข้ามาเกี่ยวข้องเฉพาะในขั้นตอนเริ่มต้นสั่งการ (ตั้งค่าที่อยู่ต้นทาง ปลายทาง และขนาดข้อมูล) และขั้นตอนสุดท้ายเมื่อการส่งผ่านข้อมูลทั้งบล็อกเสร็จสิ้น ซึ่ง DMA Controller จะส่ง Interrupt มารายงานเพียงครั้งเดียว ทำให้ CPU มีเวลาไปประมวลผลโปรเซสอื่นได้อย่างเต็มที่

    5. การจัดเก็บข้อมูลชั่วคราว (Buffering)

    Buffering คือการสำรองพื้นที่ในหน่วยความจำหลัก (RAM) เพื่อใช้พักข้อมูลชั่วคราวในระหว่างการถ่ายโอนข้อมูลระหว่างอุปกรณ์สองตัว หรือระหว่างอุปกรณ์กับโปรเซส โดยมีวัตถุประสงค์หลัก 3 ประการคือ:

    • เพื่อรองรับความเร็วที่แตกต่างกันของอุปกรณ์: เช่น การดาวน์โหลดไฟล์จากอินเทอร์เน็ตความเร็วต่ำลงสู่ฮาร์ดดิสก์ความเร็วสูง ข้อมูลจะถูกเก็บในบัฟเฟอร์ก่อนจนครบกลุ่มคำสั่งจึงจะเขียนลงดิสก์พร้อมกัน
    • เพื่อปรับขนาดบล็อกข้อมูลที่ไม่เท่ากัน: เช่น อุปกรณ์ส่งข้อมูลมาทีละไบต์ แต่ระบบปลายทางต้องการรับข้อมูลเป็นบล็อกขนาด 4KB
    • เพื่อสนับสนุนระบบเซแมนติกส์แบบไม่มีการบล็อก (Non-blocking I/O): แอปพลิเคชันสามารถส่งข้อมูลให้บัฟเฟอร์แล้วไปทำงานอื่นต่อได้ทันทีโดยไม่ต้องรอให้อุปกรณ์จริงทำงานเสร็จ

    6. การทำแคช (Caching)

    Caching เป็นเทคนิคการจัดสรรพื้นที่หน่วยความจำความเร็วสูงเพื่อเก็บสำเนาของข้อมูลจริงที่มีการใช้งานบ่อยครั้ง แตกต่างจาก Buffering ตรงที่ บัฟเฟอร์ทำหน้าที่พักข้อมูลเพื่อส่งผ่าน แต่แคชทำหน้าที่เก็บสำเนาข้อมูลเพื่อเพิ่มความเร็วในการเข้าถึงซ้ำในอนาคต (Speed up access)

    ตัวอย่างเช่น Disk Cache ซึ่งระบบปฏิบัติการจะเก็บข้อมูลบล็อกของดิสก์ที่ถูกเรียกอ่านบ่อยๆ ไว้ใน RAM เมื่อโปรเซสต้องการอ่านข้อมูลเดิมอีกครั้ง ระบบจะส่งข้อมูลจากแคชใน RAM ให้ทันที โดยไม่ต้องเสียเวลาหมุนจานแม่เหล็กหรือเข้าถึงฮาร์ดดิสก์จริง ซึ่งช่วยลดเวลาการเข้าถึงข้อมูลลงมหาศาล

    7. การสพูลลิ่ง (Spooling)

    Spooling (Simultaneous Peripheral Operations On-Line) คือเทคนิคการจัดการอุปกรณ์ I/O ที่ไม่สามารถแชร์การใช้งานพร้อมกันได้ (Non-shareable device) โดยระบบปฏิบัติการจะสร้างพื้นที่คิวพักงานไว้บนฮาร์ดดิสก์ (Spool) เพื่อรับงานจากโปรเซสต่างๆ มาเขียนเก็บลงดิสก์ก่อน แทนที่จะส่งตรงไปยังอุปกรณ์ในทันที

    ข้อดีของระบบ Spooling คือทำให้โปรเซสผู้ใช้งานทำงานเสร็จสิ้นได้อย่างรวดเร็วและสามารถทำงานอื่นต่อได้ทันที โดยไม่ต้องถูกบล็อกเพื่อรอให้อุปกรณ์ฮาร์ดแวร์อันเชื่องช้าประมวลผลเสร็จทีละงาน คิวงานบนดิสก์จะค่อยๆ ถูกทยอยส่งให้อุปกรณ์ทำงานตามลำดับอย่างเป็นระเบียบ

    8. การจัดการเครื่องพิมพ์ (Printer Management)

    เครื่องพิมพ์เป็นตัวอย่างที่ชัดเจนที่สุดของการประยุกต์ใช้ระบบ Spooling เนื่องจากเครื่องพิมพ์เป็นอุปกรณ์ประมวลผลที่ช้าและไม่สามารถจัดพิมพ์เอกสารจากสองโปรเซสพร้อมกันในเวลาเดียวกันได้ (ไม่อย่างนั้นตัวอักษรของทั้งสองไฟล์จะพิมพ์ปนกันในหน้าเดียว)

    ในระบบการจัดการเครื่องพิมพ์ เมื่อผู้ใช้กดสั่งพิมพ์เอกสาร ระบบปฏิบัติการจะแปลงเอกสารนั้นเป็นไฟล์งานพิมพ์ชั่วคราวแล้วนำไปจัดเก็บไว้ในโฟลเดอร์คิวพิมพ์ (Print Spooler) บนดิสก์ ตัวจัดการคิวพิมพ์จะเรียงลำดับงานพิมพ์และทยอยส่งข้อมูลให้เครื่องพิมพ์ทีละงานจนเสร็จสมบูรณ์ ทำให้ผู้ใช้หลายคนสามารถสั่งพิมพ์งานพร้อมกันได้โดยระบบไม่ขัดข้อง

    9. การจัดกำหนดการดิสก์ (Disk Scheduling)

    ฮาร์ดดิสก์แบบจานแม่เหล็กมีความเร็วในการทำงานต่ำกว่า CPU และ RAM มาก เนื่องจากมีส่วนประกอบที่เป็นกลไกเคลื่อนไหว การเข้าถึงข้อมูลบนดิสก์จึงใช้เวลาส่วนใหญ่ไปกับ Seek Time (เวลาที่หัวอ่านใช้เคลื่อนที่ไปยังแทร็กที่ต้องการ) ระบบปฏิบัติการจึงต้องมีอัลกอริทึมการจัดกำหนดการดิสก์ (Disk Scheduling) เพื่อจัดลำดับคำร้องขอเข้าถึงดิสก์ (I/O Requests) ใหม่ เพื่อให้หัวอ่านเคลื่อนที่สั้นที่สุดและทำงานได้เร็วที่สุด

    9.1 FCFS Disk Scheduling (First-Come, First-Served)

    เป็นอัลกอริทึมที่ง่ายที่สุด ทำงานแบบใครมาก่อนได้บริการก่อนตามลำดับคำร้องขอ ไม่มีการคำนวณตำแหน่งหัวอ่านปัจจุบัน ข้อดีคือมีความยุติธรรมและไม่มีปัญหา Starvation แต่ข้อเสียร้ายแรงคือหัวอ่านอาจต้องเคลื่อนที่สลับไปสลับมาข้ามแทร็กอย่างรุนแรง (Wild swing) ทำให้ประสิทธิภาพโดยรวมต่ำมาก

    9.2 SSTF Disk Scheduling (Shortest Seek Time First)

    อัลกอริทึมนี้จะเลือกบริการคำร้องขอที่อยู่ **ใกล้กับตำแหน่งปัจจุบันของหัวอ่านมากที่สุด** เป็นลำดับถัดไป เพื่อลดระยะทางการเคลื่อนที่ของหัวอ่านให้เหลือน้อยที่สุดในแต่ละก้าว ข้อดีคือเพิ่มประสิทธิภาพการทำงานได้ดีกว่า FCFS มาก แต่ข้อเสียคืออาจเกิดปัญหา Starvation กับคำร้องขอที่อยู่แทร็กไกลๆ หากมีคำร้องขอใหม่ๆ ป้อนเข้ามาในตำแหน่งแทร็กใกล้ๆ หัวอ่านอย่างต่อเนื่อง

    9.3 SCAN Disk Scheduling (Elevator Algorithm)

    รู้จักกันในชื่อ **"อัลกอริทึมลิฟต์"** หัวอ่านของดิสก์จะเริ่มต้นเคลื่อนที่ไปในทิศทางหนึ่ง (เช่น เคลื่อนที่เข้าสู่แทร็กในสุด) และให้บริการคำร้องขอทั้งหมดที่ขวางหน้าไปเรื่อยๆ จนกระทั่ง **เคลื่อนที่ไปถึงแทร็กปลายสุดของดิสก์** จากนั้นจึงจะเปลี่ยนทิศทางการเดินทางกลับ (เช่น เคลื่อนที่ออกไปยังแทร็กนอกสุด) และให้บริการงานที่เหลือขวางหน้าในทิศทางขากลับ วิธีนี้ช่วยแก้ปัญหา Starvation ได้อย่างสมบูรณ์

    9.4 C-SCAN Disk Scheduling (Circular SCAN)

    เป็นรูปแบบที่พัฒนาต่อยอดมาจาก SCAN เพื่อให้ระยะเวลาการรอคอยมีความสม่ำเสมอมากขึ้น โดยหัวอ่านจะเคลื่อนที่บริการคำร้องขอไปในทิศทางเดียวจนถึงปลายสุดของดิสก์ แต่เมื่อถึงจุดปลายสุดแล้ว **หัวอ่านจะเคลื่อนที่กลับมายังจุดเริ่มต้นของอีกฝั่งหนึ่งทันทีโดยไม่มีการให้บริการใดๆ ในระหว่างทางขากลับ** (กวาดข้อมูลทิศทางเดียวเป็นวงกลมต่อเนื่อง)

    10. ตัวอย่างการจัดการอุปกรณ์จริง (Real Device Management Case Study)

    ในการทำงานจริงของระบบปฏิบัติการ เช่น Linux หรือ Windows เมื่อผู้ใช้งานเชื่อมต่อแฟลชไดรฟ์ (USB Mass Storage) ระบบปฏิบัติการจะเริ่มกระบวนการจัดสรรโดยอัตโนมัติ:

    1. ตัวควบคุมพอร์ต USB ตรวจจับแรงดันไฟฟ้าที่เปลี่ยนไปและส่งสัญญาณ Interrupt แจ้ง CPU
    2. เคอร์เนลเรียกใช้งาน USB Hub Driver เพื่อระบุประเภทอุปกรณ์และโหลด USB Mass Storage Driver มารองรับ
    3. ระบบปฏิบัติการสร้างโครงสร้างย่อยระบุเป็นบล็อกอุปกรณ์ (Block Device File) เพื่อเปิดช่องทางเชื่อมโยงข้อมูล
    4. เมื่อมีการอ่านหรือคัดลอกไฟล์ขนาดใหญ่ ระบบจะสั่งเปิดกลไก DMA Channel ร่วมกับระบบ Buffering เพื่อส่งข้อมูลตรงเข้าสู่ RAM ช่วยรักษาเสถียรภาพและไม่ทำให้ CPU ทำงานหนักจนระบบค้าง

    11. แอนิเมชันจำลองขั้นตอนการขัดจังหวะด้วยสัญญาณ Interrupt และระบบ DMA

    แบบจำลองปฏิสัมพันธ์จำลองสถานการณ์ควบคุมรับส่งข้อมูล I/O จากอุปกรณ์ปลายทางผ่าน Device Controller เพื่อเข้าถึงสถาปัตยกรรมหน่วยความจำหลัก

    นักศึกษาสามารถคลิกปุ่มรันตัวจำลองเพื่อดูการทำงานของสัญญาณควบคุม:

    12. แบบจำลองเปรียบเทียบตารางแทร็กหัวอ่านการจัดกำหนดการดิสก์ (Disk Scheduling Tracing)

    พิจารณาสายลำดับคำร้องขอเข้าถึงแทร็กบนฮาร์ดดิสก์ (Disk Queue Request) ดังต่อไปนี้เพื่อจำลองตำแหน่งการขยับหัวอ่าน
