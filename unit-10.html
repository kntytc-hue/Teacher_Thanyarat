    Unit 10: Operating System Security

    หน่วยที่ 10 การรักษาความปลอดภัย

    1. ภาพรวมการรักษาความปลอดภัยของระบบปฏิบัติการ (Security Concept)

    การรักษาความปลอดภัยของระบบปฏิบัติการ (Operating System Security) หมายถึง มาตรการ นโยบาย และกลไกในการปกป้องฮาร์ดแวร์ ซอฟต์แวร์ และข้อมูลคอมพิวเตอร์จากการถูกเข้าถึง การแก้ไข การทำลาย หรือการขัดขวางโดยไม่ได้รับอนุญาต เป้าหมายหลักของการรักษาความปลอดภัยถูกกำหนดขึ้นตามหลักการพื้นฐาน CIA Triad ซึ่งประกอบด้วย:

    • Confidentiality (การรักษาความลับ): การป้องกันไม่ให้ผู้ที่ไม่ได้รับอนุญาตสามารถแอบดูหรือเข้าถึงข้อมูลสำคัญได้
    • Integrity (การรักษาความถูกต้องสมบูรณ์): การป้องกันและตรวจจับไม่ให้ข้อมูลหรือระบบถูกแก้ไข เปลี่ยนแปลง หรือทำลายโดยไม่ได้รับอนุญาต
    • Availability (การความพร้อมใช้งาน): การรับประกันว่าระบบคอมพิวเตอร์ ทรัพยากร และข้อมูล พร้อมตอบสนองและให้บริการแก่ผู้ใช้ที่มีสิทธิ์อย่างถูกต้องเมื่อมีความต้องการใช้งาน

    ระบบปฏิบัติการต้องเผชิญกับภัยคุกคามหลากหลายรูปแบบ เช่น การโจมตีทางเครือข่าย การบุกรุกโดยตรงที่ตัวเครื่อง มัลแวร์ ตลอดจนความผิดพลาดของผู้ใช้งาน (Human Error) การวางระบบความปลอดภัยจึงต้องครอบคลุมทั้งระดับไฟล์ ระบบจัดการสิทธิ์ เครือข่าย และซอฟต์แวร์ระบบ

    2. กรอบแนวคิด AAA Security Model

    กรอบการทำงานหลักที่ใช้ควบคุมการเข้าถึงทรัพยากรในระบบคอมพิวเตอร์เรียกว่า AAA Security Framework ซึ่งแบ่งออกเป็น 3 ขั้นตอนสำคัญต่อเนื่องกัน ได้แก่ Authentication, Authorization และ Accounting:

    2.1 Authentication (การยืนยันตัวตน)

    กระบวนการพิสูจน์หรือตรวจสอบว่าผู้ใช้หรือระบบที่กำลังพยายามเข้าสู่ระบบคือใคร (Who are you?) เพื่อให้มั่นใจว่าผู้ขอเข้าใช้เป็นบุคคลหรือตัวตนจริงตามที่กล่าวอ้าง โดยทั่วไปอาศัยปัจจัยพิสูจน์ (Authentication Factors) ดังนี้:

    • Something you know (สิ่งที่คุณรู้): เช่น รหัสผ่าน (Password), PIN, คำถามกันลืม
    • Something you have (สิ่งที่คุณมี): เช่น สมาร์ตการ์ด, เครื่องรับรหัส OTP, Security Key
    • Something you are (สิ่งที่คุณเป็น - Biometrics): เช่น ลายนิ้วมือ, การสแกนใบหน้า, ม่านตา
    • Multi-Factor Authentication (MFA): การใช้ปัจจัยยืนยันตัวตนมากกว่า 1 ปัจจัยร่วมกันเพื่อเพิ่มระดับความปลอดภัย

    2.2 Authorization (การอนุมัติสิทธิ์)

    กระบวนการกำหนดและตรวจสอบว่าผู้ใช้ที่ผ่านการยืนยันตัวตนแล้ว มีขอบเขตสิทธิ์ในการทำอะไรได้บ้างในระบบ (What are you allowed to do?) เช่น สิทธิ์ในการอ่าน (Read), เขียน (Write), ประมวลผล (Execute) หรือลบไฟล์ข้อมูล

    กลไกควบคุมการเข้าถึง (Access Control Lists - ACLs) และการระบุสิทธิ์ตามบทบาท (Role-Based Access Control - RBAC) ถูกนำมาใช้ในระบบปฏิบัติการ เพื่อให้มั่นใจว่าผู้ใช้แต่ละคนจะเข้าถึงเฉพาะทรัพยากรที่จำเป็นต่อการปฏิบัติงานเท่านั้น ตามหลักการสิทธิ์ต่ำสุดเท่าที่จำเป็น (Principle of Least Privilege)

    2.3 Accounting (การบันทึกและตรวจสอบประวัติ)

    กระบวนการติดตาม ติดตามผล และบันทึกกิจกรรมการใช้งานระบบของผู้ใช้ (What did you do?) เช่น เวลาที่ล็อกอิน/ล็อกเอาต์ ทรัพยากรที่เรียกใช้ การแก้ไขข้อมูล และคำสั่งที่ประมวลผล ข้อมูลบันทึกประวัติ (Log Files) เหล่านี้มีความสำคัญอย่างยิ่งต่อการตรวจสอบย้อนหลังเมื่อเกิดเหตุการณ์บุกรุก (Forensic Investigation) และการตรวจสอบการปฏิบัติตามมาตรฐานความปลอดภัย (Auditing)

    3. ไฟร์วอลล์ (Firewall)

    ไฟร์วอลล์ (Firewall) คือระบบรักษาความปลอดภัยที่ทำหน้าที่เป็นด่านหน้าในการกรองและควบคุมทราฟฟิกข้อมูลที่รับ-ส่งระหว่างเครือข่ายภายในกับเครือข่ายภายนอก (หรือระหว่างโซนต่าง ๆ ในระบบ) โดยอาศัยกฎความปลอดภัย (Security Rules) ที่กำหนดไว้

    ประเภทของไฟร์วอลล์และการทำงานหลักในระบบปฏิบัติการประกอบด้วย:

    • Packet Filtering Firewall: ตรวจสอบส่วนหัวของแพ็กเก็ตข้อมูล (เช่น IP Address, Port Number, Protocol) เพื่อตัดสินใจว่าอนุญาต (Allow) หรือปฏิเสธ (Drop/Deny)
    • Stateful Inspection Firewall: ติดตามและตรวจสอบสถานะของการเชื่อมต่อเครือข่าย (Connection State) ทำให้มีความปลอดภัยสูงกว่าการตรวจเฉพาะแพ็กเก็ตเดี่ยว
    • Host-based Firewall: ไฟร์วอลล์ระดับซอฟต์แวร์ที่ติดตั้งอยู่ในระบบปฏิบัติการเครื่องนั้นๆ โดยตรง เช่น Windows Defender Firewall หรือ ufw / iptables บน Linux เพื่อปกป้องเฉพาะเครื่องตนเอง

    4. การเข้ารหัสข้อมูล (Encryption)

    การเข้ารหัสข้อมูล (Encryption) คือกระบวนการแปลงข้อความธรรมดา (Plaintext) ให้กลายเป็นข้อความเข้ารหัส (Ciphertext) โดยใช้อัลกอริทึมทางคณิตศาสตร์และกุญแจเข้ารหัส (Encryption Key) เพื่อป้องกันไม่ให้ผู้ไม่มีกุญแจอ่านข้อมูลเข้าใจได้

    เทคโนโลยีการเข้ารหัสที่ใช้งานในระบบปฏิบัติการแบ่งออกเป็น:

    • Symmetric Encryption (การเข้ารหัสแบบสมมาตร): ใช้กุญแจดอกเดียวกันทั้งในการเข้ารหัสและถอดรหัส เช่น AES (Advanced Encryption Standard) ซึ่งมีประสิทธิภาพสูง รวดเร็ว นิยมใช้เข้ารหัสไฟล์และดิสก์
    • Asymmetric Encryption (การเข้ารหัสแบบไม่สมมาตร): ใช้กุญแจคู่กัน คือ กุญแจสาธารณะ (Public Key) สำหรับเข้ารหัส และกุญแจส่วนตัว (Private Key) สำหรับถอดรหัส เช่น RSA, ECC นิยมใช้ในกุญแจยืนยันตัวตนดิจิทัล และโปรโตคอลความปลอดภัย เช่น SSH, TLS/SSL
    • Full Disk Encryption (FDE): การเข้ารหัสข้อมูลทั้งหมดทั้งไดรฟ์รวมถึงไฟล์ระบบปฏิบัติการ เช่น BitLocker บน Windows หรือ LUKS บน Linux ช่วยปกป้องข้อมูลเมื่ออุปกรณ์ถูกโจรกรรม

    5. ซอฟต์แวร์ประสงค์ร้ายและภัยคุกคาม (Malware & Threats)

    มัลแวร์ (Malware - Malicious Software) หมายถึงโปรแกรมหรือซอฟต์แวร์ประสงค์ร้ายทุกประเภทที่ออกแบบมาเพื่อสร้างความเสียหาย รบกวน ขโมยข้อมูล หรือเข้าถึงระบบคอมพิวเตอร์โดยไม่ได้รับอนุญาต ประเภทมัลแวร์ที่สำคัญ ได้แก่:

    Virus (ไวรัสคอมพิวเตอร์)

    โปรแกรมประสงค์ร้ายที่อาศัยการเกาะติดไปกับไฟล์หรือโปรแกรมอื่น (Host File) และต้องอาศัยการสั่งประมวลผลจากผู้ใช้ (เช่น เปิดไฟล์ติดเชื้อ) เพื่อแพร่กระจายตนเองไปยังไฟล์อื่นในเครื่อง

    Worm (หนอนคอมพิวเตอร์)

    โปรแกรมประสงค์ร้ายที่สามารถคัดลอกและแพร่กระจายตัวเองผ่านระบบเครือข่ายได้โดยอัตโนมัติ ไม่ต้องอาศัยไฟล์โฮสต์หรือการปฏิสัมพันธ์ของผู้ใช้ มักสร้างปัญหาการใช้แบนด์วิดท์เครือข่ายอย่างรุนแรง

    Trojan (ม้าโทรจัน)

    โปรแกรมที่แฝงตัวมาในรูปแบบของซอฟต์แวร์ที่มีประโยชน์หรือน่าเชื่อถือ แต่แอบซ่อนฟังก์ชันประสงค์ร้ายไว้ภายใน เช่น เปิดช่องทางลับ (Backdoor) ให้ผู้บุกรุกเข้ามาควบคุมเครื่อง

    Ransomware (มัลแวร์เรียกค่าไถ่)

    มัลแวร์ที่ทำการเข้ารหัสไฟล์ข้อมูลทั้งหมดในเครื่องเหยื่อหรือบล็อกการเข้าถึงระบบ จากนั้นขู่กวาดต้อนให้ผู้ใช้จ่ายเงินค่าไถ่เพื่อแลกกับกุญแจในการถอดรหัสคืนข้อมูล

    6. ระบบรักษาความปลอดภัยในระบบปฏิบัติการเฉพาะ (OS Security Implementation)

    6.1 Windows Defender & Windows Security

    โซลูชันด้านความปลอดภัยเนทีฟที่บูรณาการอยู่ใน Windows OS มีฟังก์ชันปกป้องที่ครอบคลุมดังนี้:

    • Real-time Protection: สแกนมัลแวร์และภัยคุกคามในเวลาจริงเมื่อมีการเปิด ดาวน์โหลด หรือรันไฟล์
    • Cloud-delivered Protection: ใช้ปัญญาประดิษฐ์และฐานข้อมูลบนคลาวด์เพื่อตรวจจับภัยคุกคามชนิดใหม่ (Zero-day threats)
    • Controlled Folder Access: ป้องกันไม่ให้แอปพลิเคชันที่ไม่ได้รับอนุญาตทำการเปลี่ยนแปลงแก้ไขไฟล์ในโฟลเดอร์สำคัญ เพื่อยับยั้ง Ransomware
    • User Account Control (UAC): แจ้งเตือนและขออนุมัติจากผู้ใช้เมื่อมีโปรแกรมพยายามทำการเปลี่ยนแปลงในระดับสิทธิ์ผู้ดูแลระบบ (Administrator)

    6.2 Linux Security Mechanics

    ระบบปฏิบัติการ Linux ถูกออกแบบโครงสร้างตามหลักความปลอดภัยแบบมัลติยูสเซอร์และแยกส่วนสิทธิ์อย่างเข้มงวด:

    • File Permissions: ระบบสิทธิ์ไฟล์แบบคลาสสิก (Owner, Group, Others) ควบคุมการ Read (r), Write (w), Execute (x)
    • Root Account & Sudo: สิทธิ์บัญชี Root (Superuser) จะถูกแยกออกจากผู้ใช้ทั่วไป และใช้คำสั่ง `sudo` ในการดำเนินงานเฉพาะกิจที่ต้องใช้สิทธิ์สูง
    • Mandatory Access Control (MAC): ระบบความปลอดภัยขั้นสูงเพิ่มเติม เช่น SELinux (Security-Enhanced Linux) หรือ AppArmor เพื่อจำกัดขอบเขตการทำงานของแต่ละโปรเซสไม่ให้ละเมิดพื้นที่อื่น

    7. สำรองและการฟื้นฟูระบบ (Backup & Recovery)

    กลไกการสำรองข้อมูล (Backup) และฟื้นฟูระบบ (Recovery) คือมาตรการด่านสุดท้ายในการรับประกันความพร้อมใช้งาน (Availability) และการกู้คืนข้อมูลหากเกิดภัยพิบัติ มัลแวร์ หรือฮาร์ดแวร์ล้มเหลว

    รูปแบบการสำรองข้อมูล (Backup Types):

    • Full Backup: สำรองข้อมูลทั้งหมดที่มีในระบบ ข้อดีคือกู้คืนง่ายและรวดเร็ว ข้อเสียคือใช้พื้นที่จัดเก็บมากและใช้เวลานาน
    • Incremental Backup: สำรองเฉพาะข้อมูลที่มีการเปลี่ยนแปลงนับจากการสำรองข้อมูลครั้งล่าสุด (ไม่ว่าจะ เป็น Full หรือ Incremental) ข้อดีคือใช้พื้นที่น้อยและรวดเร็ว แต่การกู้คืนต้องใช้ไฟล์สำรองทุกชุดต่อเนื่องกัน
    • Differential Backup: สำรองเฉพาะข้อมูลที่มีการเปลี่ยนแปลงนับจากการทำ Full Backup ครั้งล่าสุด ข้อดีคือใช้เวลาและพื้นที่ปานกลาง กู้คืนได้ง่ายกว่า Incremental (ใช้ชุด Full + ชุด Differential ล่าสุด)

    กฎการสำรองข้อมูล 3-2-1 Rule: กฎมาตรฐานสากล ได้แก่ สำรองข้อมูลไว้ 3 ชุด (ชุดจริง 1 + สำรอง 2), บนสื่อจัดเก็บอย่างน้อย 2 ประเภทที่ต่างกัน, และเก็บไว้นอกสถานที่ (Off-site / Cloud) อย่างน้อย 1 ชุด

    8. Case Study: เหตุการณ์ด้านความปลอดภัยที่เกิดขึ้นจริงในโลกไซเบอร์

    กรณีศึกษาที่ 1: เหตุการณ์การแพร่ระบาดของ WannaCry Ransomware (ปี 2017)

    รายละเอียดเหตุการณ์: ในเดือนพฤษภาคม ค.ศ. 2017 เกิดการแพร่ระบาดอย่างรุนแรงของมัลแวร์เรียกค่าไถ่ชื่อ WannaCry ส่งผลกระทบต่อคอมพิวเตอร์มากกว่า 200,000 เครื่องใน 150 ประเทศทั่วโลก รวมถึงองค์กรสำคัญ เช่น ระบบบริการสุขภาพแห่งชาติของอังกฤษ (NHS)

    กลไกการโจมตี: WannaCry รวมความสามารถของ Ransomware และ Worm เข้าด้วยกัน โดยใช้ประโยชน์จากช่องโหว่การสื่อสารโปรโตคอล SMBv1 ของ Windows (รหัสช่องโหว่ EternalBlue) ทำให้สามารถแพร่กระจายผ่านเครือข่ายไปยังเครื่องอื่นได้โดยอัตโนมัติโดยที่ผู้ใช้ไม่ต้องคลิกใดๆ

    บทเรียนและการป้องกัน: การอัปเดตแพตช์ความปลอดภัยของระบบปฏิบัติการอย่างสม่ำเสมอ (Patch Management), การปิดโปรโตคอลเก่าที่ไม่ปลอดภัย (SMBv1), การสำรองข้อมูลตามกฎ 3-2-1, และการใช้ซอฟต์แวร์ป้องกันมัลแวร์ที่มีประสิทธิภาพ

    กรณีศึกษาที่ 2: การโจมตีห่วงโซ่อุปทาน SolarSun / SolarWinds Supply Chain Attack (ปี 2020)

    รายละเอียดเหตุการณ์: กลุ่มแฮกเกอร์ระดับสูงเข้าแทรกซึมระบบของบริษัท SolarWinds ผู้พัฒนาซอฟต์แวร์จัดการเครือข่าย Orion และฝังมัลแวร์ม้าโทรจัน (Trojan) ลงในซอฟต์แวร์อัปเดตที่เป็นทางการ

    กลไกการโจมตี: เมื่อลูกค้าดาวน์โหลดไฟล์อัปเดตที่เซ็นลายเซ็นดิจิทัลถูกต้อง มัลแวร์ฝังตัวจะเปิดแบล็กดอร์ (Backdoor) ส่งผลให้แฮกเกอร์สามารถเข้าถึงระบบของหน่วยงานรัฐบาลและบริษัทเทคโนโลยีชั้นนำระดับโลกจำนวนมาก

    บทเรียนและการป้องกัน: การตรวจสอบและบังคับใช้สิทธิ์ด้วยหลักการ Zero Trust Architecture, การตรวจสอบพฤติกรรมผิดปกติผ่านการจดบันทึก Log (Accounting) และการกำกับดูแลความปลอดภัยซอฟต์แวร์ภายนอกอย่างเคร่งครัด

    9. ผังกระบวนการตรวจสอบและกรองข้อมูลการเข้าถึงระบบ (Security Access Control Flow)

    แผนภาพแสดงลำดับขั้นตอนการตรวจสอบความปลอดภัย ตั้งแต่ผู้ใช้ร้องขอเข้าใช้งานผ่านไฟร์วอลล์ เข้าสู่กระบวนการยืนยันตัวตน (Authentication) ตรวจสอบสิทธิ์ (Authorization) และบันทึกกิจกรรม (Accounting):

    กระบวนการควบคุมและกรองความปลอดภัย (AAA & Firewall Flow)
    1. Request / User Input
    คำขอเข้าใช้จากผู้ใช้งานหรือเครือข่ายภายนอก
    2. Firewall Filtering
    ตรวจกรอง Packet / Port อนุญาตหรือบล็อกทราฟฟิก
    3. AAA Processing
    AuthN -> AuthZ -> Accounting (Log Event)

    11. ตัวอย่างตารางเปรียบเทียบสเปกและประเภทของภัยคุกคาม (Security Comparison)

    การสรุปและเปรียบเทียบคุณลักษณะของภัยคุกคามมัลแวร์แต่ละประเภทเพื่อการวิเคราะห์และป้องกันในระบบปฏิบัติการ:

    ประเภทมัลแวร์ รูปแบบการแพร่กระจาย ต้องการ Host File? ผลกระทบหลักต่อระบบ แนวทางป้องกันหลัก
    Virus พ่วงติดกับไฟล์โปรแกรมอื่น ต้องการ ทำลายไฟล์ แก้ไขข้อมูล ระบบทำงานผิดปกติ Antivirus, ไม่เปิดไฟล์สงสัย
    Worm แพร่ผ่านเครือข่ายอัตโนมัติ ไม่ต้องการ กินแบนด์วิดท์เครือข่าย สร้างช่องโหว่ Firewall, Patch OS อย่างสม่ำเสมอ
    Trojan หลอกลวงว่าเป็นโปรแกรมดี ไม่ต้องการ แอบเปิด Backdoor, ขโมยข้อมูลรหัสผ่าน ดาวน์โหลดจากแหล่งน่าเชื่อถือเท่านั้น
    Ransomware อีเมล Phishing, ช่องโหว่ OS แปรผัน เข้ารหัสลับไฟล์ข้อมูลเพื่อเรียกค่าไถ่ Backup 3-2-1, Defender Folder Access
