Unit 01: Introduction to Operating Systems

หน่วยที่ 1 บทนำสู่ระบบปฏิบัติการ

วัตถุประสงค์การเรียนรู้
  • อธิบายความหมายและสถาปัตยกรรมของคอมพิวเตอร์ได้อย่างถูกต้อง
  • วิเคราะห์ความแตกต่างระหว่าง Hardware, Software และ Firmware ได้อย่างแจ่มแจ้ง
  • จำแนกและเปรียบเทียบสถาปัตยกรรมและโครงสร้างของระบบปฏิบัติการแต่ละระบบในชีวิตประจำวัน
สมรรถนะรายวิชาที่พึงมี
  • แสดงความรู้เกี่ยวกับวิวัฒนาการและการประมวลผลภายในแกนกลางระบบปฏิบัติการ
  • เลือกระบบปฏิบัติการตามข้อกำหนดของคุณสมบัติงานสถาปัตยกรรมระดับอาชีวศึกษาชั้นสูง

1. ความหมายของระบบปฏิบัติการและสถาปัตยกรรมคอมพิวเตอร์

ระบบปฏิบัติการ (Operating System: OS) คือ ซอฟต์แวร์ระบบ (System Software) ที่ทำหน้าที่เป็นตัวกลางและสะพานเชื่อมโยงระดับแกนระหว่างผู้ใช้คอมพิวเตอร์ (User) แอปพลิเคชันหรือซอฟต์แวร์ประยุกต์ (Application Software) และฮาร์ดแวร์ทางกายภาพ (Hardware) หากไม่มีระบบปฏิบัติการ คอมพิวเตอร์จะเป็นเพียงกลุ่มของแผงวงจรอิเล็กทรอนิกส์ที่ไม่สามารถสื่อสารข้อมูลที่มีความหมายใดๆ กับมนุษย์ได้

ในขณะที่ สถาปัตยกรรมคอมพิวเตอร์ (Computer Architecture) หมายถึง โครงสร้าง แผนผัง และระเบียบวิธีการทำงานร่วมกันของอุปกรณ์ฮาร์ดแวร์ต่างๆ ภายในเครื่องคอมพิวเตอร์ ซึ่งประกอบไปด้วยชุดคำสั่ง (Instruction Set), การออกแบบโครงสร้างรีจิสเตอร์, และการบริหารจัดการบัสสัญญาณ โดยระบบปฏิบัติการจะเป็นผู้ควบคุมสถาปัตยกรรมเหล่านี้ให้ทำงานตามตรรกะที่สอดประสานกัน

2. ส่วนประกอบพื้นฐานของคอมพิวเตอร์: Hardware, Software, Firmware

ระบบคอมพิวเตอร์จะดำเนินงานได้อย่างสมบูรณ์ ต้องประกอบด้วยการสอดประสานของ 3 องค์ประกอบหลักดังต่อไปนี้:

  • Hardware (ฮาร์ดแวร์): ชิ้นส่วนทางกายภาพเชิงกลและอิเล็กทรอนิกส์ที่สามารถจับต้องได้ เช่น หน่วยประมวลผลกลาง (CPU), หน่วยความจำหลัก (RAM), แผงวงจรหลัก (Mainboard), และอุปกรณ์แสดงผลข้อมูล (Monitor/SSD)
  • Software (ซอฟต์แวร์): ชุดคำสั่งหรือชุดโปรแกรมที่เขียนขึ้นด้วยภาษาคอมพิวเตอร์เพื่อควบคุมให้ฮาร์ดแวร์ประมวลผลตามเป้าหมาย แบ่งออกเป็นซอฟต์แวร์ระบบ (เช่น Windows, Linux) และซอฟต์แวร์ประยุกต์ (เช่น Web Browser, Line, Office)
  • Firmware (เฟิร์มแวร์): ซอฟต์แวร์ระดับต่ำที่ถูกบันทึกและฝังอยู่อย่างถาวรในชิปหน่วยความจำประเภทไม่ลบเลือน (เช่น ROM, EEPROM) บนบอร์ดอุปกรณ์ ทำหน้าที่เสมือนตัวเริ่มเดินเครื่องและตรวจสอบฮาร์ดแวร์เบื้องต้นทันทีที่เปิดระบบ เช่น BIOS หรือ UEFI บนเมนบอร์ดคอมพิวเตอร์

3. วิวัฒนาการของระบบปฏิบัติการในแต่ละยุคสมัย

กลไกการเปลี่ยนผ่านของระบบปฏิบัติการสามารถแบ่งออกตามโครงสร้างวิศวกรรมไฟฟ้าคอมพิวเตอร์ได้ดังนี้:

3.1 ยุคแรกของคอมพิวเตอร์ (หลอดสูญญากาศ - ปลั๊กบอร์ด)

ในช่วงทศวรรษ 1945–1955 เครื่องคอมพิวเตอร์ยังไม่มีระบบปฏิบัติการ การส่งคำสั่งทำโดยการป้อนค่าทางกายภาพผ่านสายเคเบิล สวิตช์ไฟฟ้า และบัตรเจาะรู (Punch Cards) เครื่องทำงานทีละหนึ่งโปรแกรมอย่างเชื่องช้า

3.2 ระบบประมวลผลแบบกลุ่ม (Batch Processing System)

เพื่อไม่ให้ CPU อันมีราคาแพงต้องว่างงาน จึงมีการพัฒนาให้ผู้ใช้งานส่งมอบงาน (Jobs) ให้โปรแกรมควบคุมส่วนกลางทำการรวบรวมงานที่มีลักษณะรูปแบบคล้ายคลึงกัน มัดรวมกันเป็นกลุ่ม (Batch) แล้วสั่งรันต่อเนื่องกันไปโดยอัตโนมัติบนแถบบันทึกแม่เหล็ก

3.3 ระบบแบ่งส่วนเวลา (Time Sharing System)

พัฒนาขึ้นเพื่อแก้ไขปัญหาระบบ Batch โดยการใช้กลไกของ CPU เมนเฟรมสลับเวลาบริหารจัดการงานของผู้ใช้งานหลายคนผ่านเครื่องเทอร์มินอล (Terminals) ด้วยความเร็วสูงมากในเสี้ยววินาที ทำให้ผู้ใช้ทุกคนมีความรู้สึกว่าตนเองกำลังครอบครองระบบประมวลผลแต่เพียงผู้เดียว

3.4 ยุคการปฏิวัติด้วยระบบ GUI และ Personal Computer

การเปิดตัวระบบ Graphical User Interface ช่วยให้ผู้ใช้ทั่วไปเข้าถึงระบบคอมพิวเตอร์ส่วนบุคคลได้ง่ายขึ้น ผ่านการสั่งการด้วยองค์ประกอบกราฟิกอย่าง วินโดว์ หน้าต่าง ไอคอน และการคลิกเมาส์ แทนที่จะเป็นการพิมพ์ชุดคำสั่งข้อความแบบ CLI

3.5 ระบบคลาวด์ มหานครมือถือ และ AI Operating System (ปัจจุบัน - 2026)

ปัจจุบันระบบขยายตัวไปสู่ Cloud Computing และอุปกรณ์พกพาไร้สาย โดยระบบปฏิบัติการสมัยใหม่ถูกออกแบบให้มีสถาปัตยกรรมแบบกระจาย (Distributed OS) และล่าสุดในปัจจุบันคือ AI Operating System (AI-Native OS) ซึ่งรวมโครงสร้างโมเดลปัญญาประดิษฐ์เข้าไปฝังอยู่ในแกน Kernel เพื่อประมวลผลสัญญาณฮาร์ดแวร์ NPU และจัดสรรการใช้หน่วยความจำและพลังงานแบบทำนายอนาคต (Predictive Allocation)

4. ตารางวิเคราะห์เปรียบเทียบระบบปฏิบัติการยอดนิยม

ระบบปฏิบัติการ ประเภทโค้ด (Open/Closed) สถาปัตยกรรมหลัก ความปลอดภัยพื้นฐาน ความเหมาะสมกับหน้างาน
Microsoft Windows Closed Source Hybrid Kernel ปานกลาง (มี Defender) ผู้ใช้ทั่วไป, สำนักงาน, การเล่นเกม
Linux / Ubuntu Open Source Monolithic Kernel สูงมาก (เข้มงวดเรื่องสิทธิ์) เซิร์ฟเวอร์, งานเน็ตเวิร์ก, นักพัฒนา
Apple macOS Closed Source Hybrid (XNU Darwin) สูงมาก (ระบบ Sandbox) งานกราฟิกดีไซน์, พัฒนาซอฟต์แวร์
Android Open Source (AOSP) Linux Kernel Modified ปานกลางถึงสูง สมาร์ตโฟน, สมาร์ตทีวี, IoT
Apple iOS Closed Source Darwin Architecture สูงมาก (ปิดกั้นการไซด์โหลด) อุปกรณ์พกพาในเครือข่ายนิเวศ Apple

5. สื่อมัลติมีเดียประกอบบทเรียน
