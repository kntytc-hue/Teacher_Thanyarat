วิชา 21909-1012 ระบบฐานข้อมูลสำหรับงาน IoT

หน่วยที่ 5: การจัดการข้อมูลในฐานข้อมูล (CRUD Operations)

ขับเคลื่อนข้อมูลในระบบ IoT ด้วยกลุ่มคำสั่ง DML เรียนรู้การเพิ่ม ดึงข้อมูล แก้ไข และลบข้อมูลอย่างปลอดภัยและรอบคอบ

⏱️ เวลาเรียน: 2 สัปดาห์ (สัปดาห์ที่ 11 - 12 | รวมเวลา 8 ชั่วโมง)

🎯 สมรรถนะย่อยประจำหน่วยที่ 5

  • ✅ อธิบายความหมายและกลไกการทำงานของ CRUD Operations ในงานฐานข้อมูลได้
  • ✅ เขียนคำสั่ง SQL เพื่อเพิ่ม แก้ไข และลบข้อมูลได้อย่างมีไวยากรณ์และถูกต้อง
  • ✅ ตระหนักถึงความปลอดภัยและผลกระทบของการใช้คำสั่งแก้ไข/ลบข้อมูลโดยระบุเงื่อนไขอย่างรอบคอบ

💡 ทำความรู้จักกับ CRUD Operations

คำว่า CRUD คือหัวใจหลักของนักพัฒนาโปรแกรมและวิศวกร IoT ทั่วโลก เป็นคำย่อแทน 4 ธุรกรรมพื้นฐานที่กระทำต่อข้อมูลในระบบ ประกอบไปด้วยคำสั่งสากลดังนี้:

C - Createคำสั่ง INSERT
(เพิ่มข้อมูลใหม่)
R - Readคำสั่ง SELECT
(ดึงข้อมูลมาดู)
U - Updateคำสั่ง UPDATE
(แก้ไขข้อมูลเดิม)
D - Deleteคำสั่ง DELETE
(ลบข้อมูลทิ้ง)
5.1

เจาะลึกรูปแบบคำสั่งและตัวอย่างในงาน IoT

1. การเพิ่มข้อมูล (INSERT INTO)

ใช้เมื่ออุปกรณ์เซนเซอร์ (เช่น ESP32) วัดค่าได้ แล้วต้องการหย่อนค่าเหล่านั้นบันทึกลงสู่ถังเก็บฐานข้อมูล

INSERT INTO tb_sensor_logs (fk_sensor_id, temperature_value) VALUES (1, 28.5);

2. การเรียกดูข้อมูล (SELECT)

ใช้เมื่อหน้าต่างแอปพลิเคชันหรือ Dashboard ต้องการดึงข้อมูลอุณหภูมิมาแสดงผลให้มนุษย์ดูหรือพลอตกราฟ

SELECT * FROM tb_sensor_logs WHERE fk_sensor_id = 1;

3. การแก้ไขข้อมูล (UPDATE)

ใช้เมื่อช่างเทคนิคเดินไปเปลี่ยนตัวเซนเซอร์ใหม่ หรือย้ายตำแหน่งโซนติดตั้ง จึงต้องปรับปรุงประวัติระบบ

UPDATE tb_sensors SET location_zone = 'Zone_B' WHERE sensor_id = 1;

⚠️ ข้อควรระวังสูงสุด: ห้ามลืมพิมพ์เงื่อนไข WHERE เด็ดขาด! หากลืม ข้อมูลทุกตัวแปรในอาคารจะถูกเปลี่ยนเป็น Zone_B ทั้งหมด

4. การลบข้อมูล (DELETE)

ใช้เมื่อต้องการล้างข้อมูลเก่าค้างปีทิ้งเพื่อเพิ่มพื้นที่หน่วยความจำให้ระบบเซิร์ฟเวอร์

DELETE FROM tb_sensor_logs WHERE recorded_at < '2025-01-01';

⚠️ ข้อควรระวังสูงสุด: หากพิมพ์ลอยๆ แค่ DELETE FROM tb_sensor_logs; โดยไม่มี WHERE ข้อมูลล็อกประวัติเซนเซอร์ที่สะสมมาทั้งหมดจะหายวับไปในพริบตา

🛠️ 2. ลำดับกิจกรรมปฏิบัติจริง "สวมบทบาทเป็นบอร์ด IoT" (รวม 6 ชั่วโมง)

สัปดาห์ที่ 11

ฝึกหัดป้อนและเรียกดูข้อมูล (Create & Read)

นักเรียนจำลองตนเองเป็นบอร์ดควบคุม ฝึกพิมพ์คำสั่ง INSERT ข้อมูลสลับป้อนเข้าฐานข้อมูล และใช้คำสั่ง SELECT กรองเจาะจงเฉพาะช่วงค่าที่ครูกำหนด

สัปดาห์ที่ 12

ปฏิบัติการปรับปรุงและล้างขยะ (Update & Delete)

ฝึกทำภารกิจศัลยกรรมข้อมูลช่างเทคนิคย้ายระบบ ฝึกพิมพ์ UPDATE และ DELETE อย่างรอบคอบ โดยตรวจสอบการทำงานของเงื่อนไข WHERE ทุกครั้งก่อนกดรันคำสั่งจริง

📝 ใบงานออนไลน์ที่ 5: แบบทดสอบไวยากรณ์และความปลอดภัย CRUD

ฝึกฝนทักษะการวิเคราะห์ไวยากรณ์ DML ผ่านระบบถามตอบอินเทอร์แอคทีฟ

❔ โจทย์ข้อที่ 1: รถขนส่งรหัสทะเบียน "กข-777" ต้องการบันทึกประวัติความเร็วใหม่เข้าตารางในฐานข้อมูล ต้องใช้คำสั่งหลักใดในกลุ่ม CRUD?

ตอบ: คำสั่ง INSERT INTO (ธุรกรรมด่าน Create) เพื่อเอาแถวข้อมูลชุดความเร็วชุดใหม่ไปหย่อนลงตาราง

❔ โจทย์ข้อที่ 2: อะไรคือความเสี่ยงที่ร้ายแรงที่สุดหากโปรแกรมเมอร์เผลอรันคำสั่ง DELETE FROM tb_users; โดยลืมใส่ส่วนของเงื่อนไข?

ตอบ: ข้อมูลผู้ใช้งานทั้งหมดในตารางจะถูกล้างทิ้งจนเกลี้ยงเตียนทันที เนื่องจากไม่มีประโยคเงื่อนไขกำกับว่าให้ลบแถวไหน ระบบจึงดำเนินการกวาดล้างทุกแถวที่มีอยู่ในโครงสร้างตาราง

❔ โจทย์ข้อที่ 3: หากต้องการอัปเดตสถานะของอุปกรณ์เซนเซอร์รหัส ID หมายเลข 5 ให้เป็นสถานะ 'OFFLINE' ต้องพิมพ์ไวยากรณ์แบบใด?

ตอบ: UPDATE tb_sensors SET device_status = 'OFFLINE' WHERE sensor_id = 5;
