5.1 ความหมายของโปรโตคอล (Protocol Definition)

ในระบบเครือข่ายคอมพิวเตอร์ โปรโตคอลคือ "ข้อตกลง กฎระเบียบ หรือภาษากลาง" ที่กำหนดขึ้นเพื่อควบคุมสถาปัตยกรรมการส่งข้อมูลระหว่างอุปกรณ์อิเล็กทรอนิกส์ หากไม่มีโปรโตคอล คอมพิวเตอร์ต่างผู้ผลิตหรือคนละระบบปฏิบัติการจะไม่สามารถแลกเปลี่ยนข้อมูลกันได้อย่างถูกต้อง

💡 ตัวอย่างเปรียบเทียบเชิงวิชาชีพ: เหมือนการขับรถบนท้องถนนที่ต้องมี "กฎจราจร" ร่วมกัน (เช่น การเปิดไฟเลี้ยว, การหยุดรถตรงสัญญาณไฟ) หากทุกคนไม่ทำตามกฎเดียวกัน ถนนก็จะไม่สามารถสัญจรได้

5.2 ชุดโปรโตคอล TCP/IP (Transmission Control Protocol / Internet Protocol)

นี่คือมาตรฐานโครงสร้างหลักของอินเทอร์เน็ตในปัจจุบัน ทำหน้าที่จัดการการส่งผ่านแพ็กเก็ตข้อมูลจากจุดเริ่มต้นไปยังปลายทาง โดยทำงานร่วมกันเป็นสถาปัตยกรรมแบบเลเยอร์ (Layer)

📌 TCP (Connection-Oriented)
  • รับประกันความถูกต้องของข้อมูล (Reliable)
  • มีกลไกตรวจสอบข้อผิดพลาด (Error Checking) และกระบวนการ Three-way Handshake
  • หากข้อมูลสูญหายจะเกิดการขอส่งใหม่ (Retransmission)
📌 IP (Internet Protocol)
  • ทำหน้าที่ระบุที่อยู่ของอุปกรณ์ปลายทาง (IP Addressing)
  • จัดการการเส้นทางของ Packet (Routing)
  • เน้นการค้นหาเส้นทางที่ดีที่สุด แต่ไม่รับประกันความปลอดภัยของกล่องข้อมูล

5.3 โปรโตคอล FTP (File Transfer Protocol)

โปรโตคอลสำหรับการโอนย้ายไฟล์งานคอมพิวเตอร์ ข้อมูลรูปภาพ หรือซอร์สโค้ดโปรแกรมผ่านเครือข่าย ทำงานในรูปแบบ Client-Server โดยใช้สถาปัตยกรรมพอร์ตคู่: Port 21 สำหรับส่งคำสั่งควบคุม (Control) และ Port 20 สำหรับส่งข้อมูลไฟล์จริง (Data)

5.4 โปรโตคอล HTTP และ HTTPS (HyperText Transfer Protocol)

โปรโตคอลที่ขับเคลื่อนระบบเวิลด์ไวด์เว็บ (WWW) ทำงานบนพอร์ตมาตรฐาน Port 80 โดยเป็นการแลกเปลี่ยนเอกสารเว็บในรูปแบบ Hypertext (HTML/CSS/JS)

🛡️ เพิ่มเติมเพื่อความทันสมัย: ปัจจุบันนิยมเปลี่ยนมาใช้ HTTPS (Port 443) ซึ่งใช้กระบวนการเข้ารหัสลับ SSL/TLS เพื่อป้องกันการดักจับข้อมูล (Eavesdropping) ของมิจฉาชีพ

5.5 โปรโตคอล UDP (User Datagram Protocol)

ทางเลือกทดแทน TCP ในจุดที่ต้องการความเร็วขั้นสูงสุด ทำงานในลักษณะ Connectionless คือส่งข้อมูลออกไปทันทีโดยไม่ตรวจเช็คว่าปลายทางพร้อมรับหรือไม่ และไม่สนใจว่าข้อมูลจะตกหล่นระหว่างทางหรือไม่ เหมาะกับระบบสตรีมมิ่ง วิดีโอคอล หรือเกมออนไลน์

📝 ใบงานที่ 8: กิจกรรมตรวจสอบความเข้าใจโปรโตคอล

เก็บคะแนนทฤษฎี 10 คะแนน

คำชี้แจง: ให้นักเรียนวิเคราะห์สถานการณ์ต่อไปนี้ แล้วตอบว่าควรเลือกใช้โปรโตคอลใด พร้อมระบุเหตุผลเชิงเทคนิค

  1. นักเรียนต้องการทำระบบ Video Streaming การถ่ายทอดสดคอนเสิร์ตวงดนตรีผ่านเว็บ -> [ ตอบ UDP เพราะ...]
  2. เจ้าหน้าที่ฝ่ายบัญชีต้องการส่งไฟล์สรุปงบประมาณประจำปีขนาด 500MB ขึ้นระบบคลาวด์ -> [ ตอบ FTP บน TCP เพราะ...]
  3. ลูกค้าธนาคารต้องการเข้าสู่ระบบ Mobile Banking เพื่อทำธุรกรรมทางการเงิน -> [ ตอบ HTTPS พอร์ต 443 เพราะ...]

ปฏิบัติการห้องแล็บคอมพิวเตอร์ (120 นาที)

ให้นักเรียนเปิดเครื่องคอมพิวเตอร์และลงมือปฏิบัติงานรายบุคคลตามโครงสร้างขั้นตอนด้านล่างนี้

ปฏิบัติการหลัก

ใบงานที่ 9: การประยุกต์ใช้งานและทดสอบโปรแกรม FTP Client/Server

🛠️ เครื่องมือที่ต้องใช้: 1. คอมพิวเตอร์ระบบ LAN
2. โปรแกรม Filezilla Client
3. บัญชี FTP Sandbox ของห้องเรียน
🎯 วัตถุประสงค์: เพื่อให้นักเรียนเข้าใจกระบวนการทำงานของ Port 20, 21 และรับส่งไฟล์ผ่านสถาปัตยกรรมอินเทอร์เน็ตได้จริง
💯 เกณฑ์คะแนน (10 คะแนน): - เชื่อมต่อสำเร็จ (3)
- อัปโหลด/ดาวน์โหลดถูก (4)
- สรุปผลถูกต้องวิชาการ (3)

🚀 ขั้นตอนการทดลองปฏิบัติงาน

1

ตรวจสอบและเปิดโปรแกรม

ให้นักเรียนดับเบิ้ลคลิกเพื่อเปิดใช้งานโปรแกรม FileZilla Client บนหน้าจอ Desktop ของตนเอง

2

ป้อนค่าพารามิเตอร์ของสถานีปลายทาง (Host Connection)

กรอกข้อมูลการจำลองเครือข่ายตามที่คุณครูกำหนดไว้ที่หน้ากระดานดำ:

Host: 192.168.x.x (ตามเบอร์ Server เครื่องครู)
Username: std_pvoc2
Password: password2026
Port: 21
3

ทดสอบระบบการโอนย้ายไฟล์ (Data Transfer)

ให้ลากไฟล์รูปภาพในเครื่องของนักเรียนฝั่งซ้าย (Local Site) ข้ามไปยังฝั่งขวา (Remote Site) เพื่อทดสอบการ Upload จากนั้นสังเกตกล่องข้อความสถานะด้านบนสุดว่าขึ้นคำว่า "Transfer successful" หรือไม่
