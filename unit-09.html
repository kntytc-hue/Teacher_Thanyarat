    Unit 09: File Systems & Storage Management

    หน่วยที่ 9 ระบบแฟ้ม

    1. ระบบแฟ้ม (File System)

    ระบบแฟ้ม (File System) คือ โครงสร้างและกลไกซอฟต์แวร์ที่ระบบปฏิบัติการใช้ในการจัดเก็บ จัดระเบียบ ค้นหา และเข้าถึงข้อมูลบนอุปกรณ์จัดเก็บข้อมูลสำรอง (Secondary Storage) เช่น ฮาร์ดดิสก์ (HDD), โซลิดสเตตไดรฟ์ (SSD), แฟลชไดรฟ์ หรือออปติคัลดิสก์ หากปราศจากระบบแฟ้ม ข้อมูลที่จัดเก็บไว้จะกลายเป็นเพียงลำดับของไบต์ (Stream of bytes) ที่ต่อเนื่องกันโดยไม่มีจุดเริ่มต้นหรือจุดสิ้นสุดที่ชัดเจน ทำให้ไม่สามารถระบุได้ว่าข้อมูลชุดใดเป็นของโปรแกรมใด

    ระบบแฟ้มทำหน้าที่เป็นชั้นของการนามธรรม (Abstraction Layer) ระหว่างผู้ใช้หรือแอปพลิเคชันกับอุปกรณ์จัดเก็บข้อมูลทางกายภาพ ช่วยแปลงคำสั่งระดับสูง เช่น การสร้าง อ่าน แก้ไข หรือลบไฟล์ ให้กลายเป็นการทำงานระดับบล็อก (Block-level I/O Operations) บนอุปกรณ์จัดเก็บข้อมูลจริง นอกจากนี้ ระบบแฟ้มยังรับผิดชอบการบริหารจัดการพื้นที่ว่าง (Free Space Management) การบันทึกตำแหน่งบล็อกของไฟล์ และการรักษาความปลอดภัยของข้อมูล

    2. แฟ้มข้อมูล (File)

    แฟ้มข้อมูล หรือ ไฟล์ (File) คือ หน่วยของการจัดเก็บข้อมูลเชิงตรรกะ (Logical Storage Unit) ที่ถูกกำหนดขึ้นโดยระบบปฏิบัติการ เพื่อรวบรวมกลุ่มของข้อมูลที่มีความสัมพันธ์กันไว้ด้วยกัน ภายใต้ชื่อที่กำหนดขึ้น จากมุมมองของผู้ใช้ ไฟล์เป็นหน่วยข้อมูลขนาดเล็กที่สุดที่สามารถบันทึกลงในระบบจัดเก็บข้อมูลแบบถาวรได้

    ไฟล์สามารถแบ่งออกได้เป็นหลายประเภทตามลักษณะของข้อมูล เช่น

    • Executable Files (.exe, .bin, .sh): ไฟล์รหัสคำสั่งที่ภาษาเครื่องสามารถนำไปประมวลผลได้โดยตรง
    • Data Files (.txt, .csv, .json): ไฟล์ข้อมูลข้อความเปล่าหรือข้อมูลโครงสร้าง
    • Document Files (.docx, .pdf, .xlsx): ไฟล์เอกสารที่มีรูปแบบเฉพาะ
    • Multimedia Files (.jpg, .mp3, .mp4): ไฟล์สื่อภาพ เสียง และวิดีโอ

    โครงสร้างภายในไฟล์ถูกตีความโดยแอปพลิเคชันหรือระบบปฏิบัติการ โดยมีการเข้าถึงข้อมูลในไฟล์ 2 วิธีหลัก คือ การเข้าถึงแบบลำดับ (Sequential Access) ซึ่งเป็นการอ่านหรือเขียนข้อมูลเรียงตามลำดับไบต์ตั้งแต่นับเริ่มต้นจนถึงจุดสิ้นสุด และการเข้าถึงแบบสุ่มหรือโดยตรง (Direct/Random Access) ซึ่งช่วยให้โปรแกรมสามารถข้ามไปยังตำแหน่งใดๆ ของไฟล์ได้ทันทีโดยไม่ต้องอ่านข้อมูลก่อนหน้า

    3. สารบบ (Directory)

    สารบบ หรือ ไดเรกทอรี (Directory / Folder) คือ โครงสร้างข้อมูลทางตรรกะที่ทำหน้าที่จัดกลุ่มและบันทึกรายชื่อของแฟ้มข้อมูล เพื่อให้ผู้ใช้และระบบปฏิบัติการสามารถค้นหาและจัดหมวดหมู่ไฟล์ได้อย่างเป็นระบบ ในระดับล่าง ไดเรกทอรีก็คือไฟล์ชนิดพิเศษประเภทหนึ่งที่ภายในบรรจุตารางรายการ (Directory Entries) ซึ่งเชื่อมโยงระหว่างชื่อไฟล์กับตำแหน่งทางกายภาพหรือดัชนีระบุตัวตนของไฟล์นั้นๆ บนดิสก์ (เช่น Inode Number หรือ FAT Cluster)

    การใช้งานไดเรกทอรีช่วยแก้ปัญหาความสับสนเมื่อมีจำนวนไฟล์เพิ่มมากขึ้น รองรับการตั้งชื่อไฟล์ที่ซ้ำกันหากอยู่นอกไดเรกทอรีเดียวกัน และช่วยอำนวยความสะดวกในการกำหนดสิทธิ์การเข้าถึงไฟล์แบบยกกลุ่ม

    4. โครงสร้างสารบบแบบลำดับชั้น (Directory Hierarchy)

    โครงสร้างสารบบแบบลำดับชั้น (Directory Hierarchy หรือ Tree-Structured Directory) เป็นรูปแบบการจัดหมวดหมู่สารบบที่นิยมใช้งานมากที่สุดในระบบปฏิบัติการสมัยใหม่ โดยมีลักษณะเป็นโครงสร้างต้นไม้ที่มี Root Directory เป็นจุดเริ่มต้นหรือโคนต้น แล้วแตกกิ่งก้านสาขาออกเป็น Subdirectories และ Files ในชั้นถัดๆ ไป

    การอ้างอิงตำแหน่งของไฟล์หรือไดเรกทอรีภายในโครงสร้างแบบลำดับชั้น แบ่งออกเป็น 2 ประเภทหลัก:

    • Path แบบสัมบูรณ์ (Absolute Path): การระบุเส้นทางเริ่มต้นตั้งแต่ Root Directory ไปจนถึงไฟล์ปลายทาง เช่น C:\Users\Student\Documents\unit-09.html บน Windows หรือ /var/log/syslog บน Linux
    • Path แบบสัมพัทธ์ (Relative Path): การระบุเส้นทางโดยอ้างอิงจากตำแหน่ง Working Directory ในปัจจุบัน เช่น ./images/photo.png หรือ ../unit-08.html

    5. ระบบแฟ้ม FAT16

    FAT16 (File Allocation Table 16-bit) เป็นระบบแฟ้มที่ถูกพัฒนาโดย Microsoft ในยุค MS-DOS และ Windows 3.x/95 โดยใช้ตาราง FAT ขนาด 16 บิต ในการดักจับและอ้างอิงตำแหน่งของ Cluster บนดิสก์ ส่งผลให้สามารถอ้างอิง Cluster ได้สูงสุด 216 = 65,536 Clusters

    ข้อจำกัดของ FAT16 คือ รองรับขนาดพาร์ทิชันสูงสุดเพียง 2 GB (หรือ 4 GB ในบางกรณีพิเศษที่ใช้ขนาด Cluster 64 KB) และมีประสิทธิภาพต่ำเมื่อใช้กับพาร์ทิชันขนาดใหญ่ เนื่องจากก่อให้เกิดความสูญเสียพื้นที่ภายในบล็อก (Internal Fragmentation) สูง นอกจากนี้ FAT16 ยังไม่มีระบบรักษาความปลอดภัยระดับไฟล์และการทำ Journaling

    6. ระบบแฟ้ม FAT32

    FAT32 เป็นระบบแฟ้มที่ถูกพัฒนาต่อยอดจาก FAT16 โดยเปิดตัวพร้อมกับ Windows 95 OSR2 ใช้การอ้างอิงตำแหน่งด้วยความยาว 28 บิต (จากขนาดฟิลด์ 32 บิต) ทำให้รองรับจำนวน Cluster ได้สูงสุดถึง 268,435,456 Clusters ส่งผลให้รองรับขนาดพาร์ทิชันได้สูงสุดถึง 2 TB (หรือ 16 TB ในทางทฤษฎี)

    แม้ FAT32 จะเพิ่มประสิทธิภาพในการใช้พื้นที่ดิสก์และขยายขนาดพาร์ทิชันได้อย่างมาก แต่มีข้อจำกัดวิกฤตคือ **ไม่สามารถเก็บไฟล์เดี่ยวที่มีขนาดใหญ่เกิน 4 GB (4,294,967,295 ไบต์)** ได้ อย่างไรก็ตาม FAT32 มีจุดเด่นสำคัญด้านความสามารถในการทำงานข้ามระบบปฏิบัติการ (Cross-platform Compatibility) ทำให้ยังนิยมใช้ในอุปกรณ์บันทึกข้อมูลพกพาขนาดเล็ก

    7. ระบบแฟ้ม NTFS

    NTFS (New Technology File System) เป็นระบบแฟ้มหลักของระบบปฏิบัติการ Windows ตระกูล NT (Windows NT, 2000, XP, Vista, 7, 8, 10, 11, และ Windows Server) ที่ถูกออกแบบมาเพื่อทดแทน FAT32 โดยมุ่งเน้นประสิทธิภาพ ความน่าเชื่อถือ และความปลอดภัยสูง

    ฟีเจอร์สำคัญของ NTFS ได้แก่:

    • Journaling: บันทึกประวัติการเปลี่ยนแปลงลงใน Log ($LogFile) ก่อนเขียนข้อมูลจริง ช่วยให้ระบบสามารถกู้คืนสภาวะปกติได้อย่างรวดเร็วเมื่อเกิดเหตุไฟดับหรือระบบล่ม
    • Access Control List (ACL): กำหนดสิทธิ์การเข้าถึงไฟล์และโฟลเดอร์ในระดับผู้ใช้และกลุ่มผู้ใช้ได้อย่างละเอียด
    • File Encryption & Compression: รองรับการบีบอัดข้อมูล (NTFS Compression) และการเข้ารหัสไฟล์ (EFS - Encrypting File System)
    • MFT (Master File Table): ตารางโครงสร้างหลักที่ใช้เก็บบันทึกข้อมูล Metadata ของไฟล์และไดเรกทอรีทั้งหมดในพาร์ทิชัน

    8. ระบบแฟ้ม exFAT

    exFAT (Extended File Allocation Table) เป็นระบบแฟ้มที่ Microsoft ออกแบบมาโดยเฉพาะสำหรับอุปกรณ์จัดเก็บข้อมูลแบบแฟลช (Flash Storage) เช่น แฟลชไดรฟ์, การ์ด SDXC และ SSD พกพา เพื่อข้ามขีดจำกัดของ FAT32 โดยไม่สร้างภาระการทำงาน (Overhead) สูงเหมือน NTFS

    exFAT ก้าวข้ามข้อจำกัดขนาดไฟล์ 4 GB ของ FAT32 โดยรองรับขนาดไฟล์เดี่ยวได้สูงสุดถึง 16 EB (Exabytes) และรองรับขนาดพาร์ทิชันสูงสุด 128 PB (Petabytes) นอกจากนี้ exFAT ยังได้รับการออกแบบให้รองรับการอ่านและเขียนข้อมูลข้ามระบบปฏิบัติการ ทั้ง Windows, macOS, Linux, Android และ iOS ได้อย่างสมบูรณ์โดยไม่ต้องติดตั้งไดรเวอร์เพิ่มเติม

    9. ระบบแฟ้ม EXT3

    EXT3 (Third Extended Filesystem) เป็นระบบแฟ้มบันทึกการเปลี่ยนแปลง (Journaling File System) ที่พัฒนาต่อยอดมาจาก EXT2 สำหรับระบบปฏิบัติการ Linux นำเสนอความสามารถในการบันทึก Log การทำงาน ช่วยลดเวลาในการตรวจสอบและแก้ไขความผิดพลาดของดิสก์ (fsck) หลังจากระบบหยุดทำงานอย่างผิดปกติ

    EXT3 รองรับขนาดพาร์ทิชันสูงสุด 2 TB ถึง 16 TB (ขึ้นอยู่กับขนาดบล็อก) และขนาดไฟล์เดี่ยวสูงสุด 16 GB ถึง 2 TB โหมดการทำ Journaling ของ EXT3 แบ่งออกเป็น 3 ระดับ คือ Journal (เข้า Log ทั้งข้อมูลและ Metadata), Ordered (เข้า Log เฉพาะ Metadata แต่เขียนข้อมูลจริงก่อน) และ Writeback (เข้า Log เฉพาะ Metadata โดยไม่การันตีลำดับเขียนข้อมูล)

    10. ระบบแฟ้ม EXT4

    EXT4 (Fourth Extended Filesystem) เป็นระบบแฟ้มมาตรฐานที่ถูกใช้งานอย่างแพร่หลายใน Linux ดิสโทรสมัยใหม่ พัฒนาขึ้นเพื่อทดแทน EXT3 โดยมีการปรับปรุงโครงสร้างเพื่อเพิ่มประสิทธิภาพ ขยายความจุ และลดปัญหาความกระจัดกระจายของข้อมูล (Fragmentation)

    คุณสมบัติเด่นของ EXT4 ประกอบด้วย:

    • Large File System Support: รองรับพาร์ทิชันขนาดใหญ่สูงสุด 1 EB และไฟล์เดี่ยวขนาดสูงสุด 16 TB
    • Extents: เปลี่ยนรูปแบบการจัดเก็บตำแหน่งบล็อกจากการอ้างอิงทีละบล็อกเป็นการอ้างอิงช่วงบล็อกที่ต่อเนื่องกัน (Contiguous Blocks) ช่วยเพิ่มประสิทธิภาพการอ่านเขียนไฟล์ขนาดใหญ่
    • Delayed Allocation: ชะลอการจัดสรรบล็อกไว้จนกว่าจะมีการเขียนข้อมูลจริงลงดิสก์ ช่วยลด Fragmentation และเพิ่มประสิทธิภาพ I/O
    • Fast fsck: ข้ามการตรวจสอบบล็อกที่ไม่ถูกใช้งาน ทำให้ทำการตรวจสอบดิสก์ได้เร็วขึ้นอย่างมาก

    11. ระบบแฟ้ม Btrfs

    Btrfs (B-tree File System) เป็นระบบแฟ้มรุ่นใหม่สำหรับ Linux ที่ออกแบบตามหลักการ Copy-on-Write (CoW) มุ่งเน้นการรองรับระบบจัดเก็บข้อมูลขนาดใหญ่ ความแม่นยำของข้อมูล และความคล่องตัวในการบริหารจัดการ จัดเป็นระบบแฟ้มระดับสูงที่รวมเอาความสามารถของ Volume Manager เข้ามาไว้ในตัว

    จุดเด่นของ Btrfs ได้แก่ การทำ Snapshot ชั่วขณะได้ทันทีโดยไม่เปลืองพื้นที่ (Writable/Read-only Snapshots), การตรวจสอบและแก้ไขความสูญเสียของข้อมูลอัตโนมัติด้วย Checksum (Data Integrity & Self-healing), การทำ RAID ระดับซอฟต์แวร์ในตัว (RAID 0, 1, 10), การบีบอัดข้อมูลแบบเรียลไทม์ (zlib, lzo, zstd) และการขยายหรือลดขนาดพาร์ทิชันขณะระบบทำงาน (Online Resizing)

    12. ระบบแฟ้ม XFS

    XFS เป็นระบบแฟ้มแบบ Journaling 64 บิต ประสิทธิภาพสูง ที่พัฒนาขึ้นโดยบริษัท Silicon Graphics International (SGI) และถูกนำมาใช้เป็นระบบแฟ้มมาตรฐานหลักใน Enterprise Linux หลายตระกูล เช่น Red Hat Enterprise Linux (RHEL) และ CentOS

    XFS ถูกออกแบบมาเป็นพิเศษสำหรับการประมวลผลข้อมูลขนาดมหาศาล (Parallel I/O) รองรับระบบไฟล์และไฟล์เดี่ยวขนาดใหญ่สูงสุดถึง 8 EB การจัดการโครงสร้างภายในอาศัย B+ trees เพื่อควบคุมการจัดสรรบล็อกและ Inode ส่งผลให้คงประสิทธิภาพการทำงานที่รวดเร็วสูงแม้ต้องรับมือกับไฟล์ขนาดใหญ่หลายล้านไฟล์พร้อมกัน

    13. สิทธิ์การเข้าถึงแฟ้มข้อมูล (Permission)

    สิทธิ์การเข้าถึง (Permission) คือ กลไกความปลอดภัยขั้นพื้นฐานที่ใช้ควบคุมว่าผู้ใช้ (User) หรือกระบวนการ (Process) ใด สามารถกระทำกิจกรรมประเภทใดกับไฟล์หรือไดเรกทอรีได้บ้าง เพื่อป้องกันการแก้ไข หรือเข้าถึงข้อมูลโดยไม่ได้รับอนุญาต

    ในระบบปฏิบัติการ Linux/Unix สิทธิ์การเข้าถึงแบ่งออกเป็น 3 สิทธิ์พื้นฐาน:

    • Read (r - สิทธิ์ในการอ่าน): สำหรับไฟล์ คือการอ่านเนื้อหาภายใน; สำหรับไดเรกทอรี คือการดูรายชื่อไฟล์ภายในสารบบ
    • Write (w - สิทธิ์ในการเขียน): สำหรับไฟล์ คือการแก้ไข บันทึก ทับเนื้อหา; สำหรับไดเรกทอรี คือการสร้าง ลบ หรือเปลี่ยนชื่อไฟล์ในสารบบ
    • Execute (x - สิทธิ์ในการประมวลผล): สำหรับไฟล์ คือการรันโปรแกรมหรือสคริปต์; สำหรับไดเรกทอรี คือการเข้าไปข้างในสารบบ (cd)

    สิทธิ์ถูกจัดกลุ่มตามผู้ใช้งาน 3 ระดับ คือ เจ้าของแฟ้ม (Owner/User - u), กลุ่มผู้ใช้ (Group - g), และผู้ใช้อื่นๆ (Others - o) การกำหนดสิทธิ์สามารถแสดงได้ทั้งรูปแบบสัญลักษณ์ (เช่น rwxr-xr--) และรูปแบบตัวเลขฐานแปด (Octal Notation เช่น 754)

    14. รายการควบคุมการเข้าถึง (ACL)

    รายการควบคุมการเข้าถึง หรือ ACL (Access Control List) คือ กลไกการกำหนดสิทธิ์ความปลอดภัยส่วนขยาย ที่ถูกออกแบบมาเพื่อทลายข้อจำกัดของการกำหนดสิทธิ์แบบพื้นฐาน (Owner/Group/Others) ช่วยให้ผู้ดูแลระบบสามารถกำหนดสิทธิ์ในการเข้าถึงไฟล์หรือไดเรกทอรีให้กับผู้ใช้เฉพาะราย (Specific User) หรือกลุ่มผู้ใช้เฉพาะกลุ่ม (Specific Group) ได้อย่างอิสระและยืดหยุ่นสูง

    ตัวอย่างเช่น กรณีต้องการให้ผู้ใช้ A อ่านและแก้ไขไฟล์ได้ ให้ผู้ใช้ B อ่านได้อย่างเดียว แต่ผู้ใช้อื่นๆ ในกลุ่มเดียวกันไม่มีสิทธิ์เข้าถึงเลย กลไก Permission พื้นฐานจะไม่สามารถรองรับได้ แต่สามารถจัดการได้ง่ายผ่านคำสั่ง ACL บน NTFS (Windows) หรือ POSIX ACLs บน Linux (เช่น คำสั่ง setfacl และ getfacl)

    15. ข้อมูลอธิบายแฟ้มข้อมูล (File Metadata)

    ข้อมูลอธิบายแฟ้มข้อมูล หรือ ไฟล์เมทาเดตา (File Metadata) คือ "ข้อมูลที่ใช้อธิบายข้อมูล" หมายถึงรายละเอียดคุณลักษณะและคุณสมบัติต่างๆ ของแฟ้มข้อมูล ซึ่งถูกบันทึกและจัดเก็บไว้แยกต่างหากจากเนื้อหาจริงของไฟล์ (Data Blocks) โดยในระบบ Linux ข้อมูลเมทาเดตาจะถูกจัดเก็บไว้ในโครงสร้างที่เรียกว่า **Inode (Index Node)** ส่วนใน NTFS จะถูกเก็บบันทึกไว้ในตาราง **MFT (Master File Table)**

    องค์ประกอบสำคัญของ File Metadata ประกอบด้วย:

    รายการ Metadata คำอธิบาย
    File Size ขนาดของไฟล์ (แสดงในหน่วย Bytes, KB, MB)
    File Ownership ข้อมูลระบุผู้เป็นเจ้าของไฟล์ (User ID / Group ID)
    Access Permissions การกำหนดสิทธิ์การเข้าถึง (rwx / ACLs)
    Timestamps เวลาในการเข้าถึงล่าสุด (atime), เวลาในการแก้ไขเนื้อหาล่าสุด (mtime), และเวลาในการเปลี่ยนสถานะ/Metadata ล่าสุด (ctime)
    Block Pointers ตำแหน่งตัวชี้พิกัดบล็อกข้อมูลจริงบนอุปกรณ์จัดเก็บข้อมูล

    16. ตัวอย่างการจัดการไฟล์ใน Windows และ Linux

    การจัดการไฟล์ผ่านบรรทัดคำสั่ง (Command Line Interface - CLI) เป็นทักษะสำคัญสำหรับผู้ดูแลระบบ ทั้งในสภาพแวดล้อม Windows (Command Prompt / PowerShell) และ Linux (Terminal / Bash) ตัวอย่างเปรียบเทียบคำสั่งการจัดการไฟล์พื้นฐานแสดงดังตารางต่อไปนี้:

    การทำงาน Windows (CMD / PowerShell) Linux (Bash Shell)
    แสดงรายชื่อไฟล์/โฟลเดอร์ dir / Get-ChildItem ls -l
    เปลี่ยนสารบบปัจจุบัน cd path\to\dir cd /path/to/dir
    สร้างสารบบใหม่ mkdir new_folder mkdir new_folder
    สร้างไฟล์ข้อความใหม่ type nul > file.txt touch file.txt
    คัดลอกไฟล์ copy file1.txt file2.txt cp file1.txt file2.txt
    ย้ายหรือเปลี่ยนชื่อไฟล์ move old.txt new.txt mv old.txt new.txt
    ลบไฟล์ del file.txt rm file.txt
    ตรวจสอบสิทธิ์และการกำหนดสิทธิ์ icacls file.txt chmod 755 file.txt / chown user:group file.txt
    โครงสร้างลำดับชั้นระบบแฟ้มข้อมูล (Directory Hierarchy Overview)
    Windows Hierarchy (Drive Letters)
    C:\ (System Drive) -> Program Files, Users, Windows
    Linux Hierarchy (Single Tree Root)
    / (Root) -> /bin, /etc, /home, /var, /usr

    17. Animation จำลองการทำงานของระบบแฟ้มและการเข้าถึงข้อมูล

    18. ตัวอย่างสคริปต์การจัดการสิทธิ์และโครงสร้างแฟ้มข้อมูล

    ตัวอย่างการใช้คำสั่งบน Linux Terminal เพื่อสร้างไดเรกทอรี สร้างไฟล์ ตรวจสอบ Metadata และกำหนดสิทธิ์การเข้าถึง:

    # 1. สร้างโครงสร้างไดเรกทอรีแบบลำดับชั้น mkdir -p /home/student/os_project/unit09 # 2. สร้างไฟล์และบันทึกข้อความ echo "File System & Storage Management" > /home/student/os_project/unit09/notes.txt # 3. แสดงข้อมูล File Metadata และ Inode Number ls -li /home/student/os_project/unit09/notes.txt # 4. กำหนดสิทธิ์แบบ Octal (Owner=Read/Write/Exec, Group=Read/Exec, Others=Read) chmod 755 /home/student/os_project/unit09/notes.txt # 5. กำหนดสิทธิ์เพิ่มเติมด้วย ACL ให้ผู้ใช้ 'teacher' มีสิทธิ์อ่านเขียน setfacl -m u:teacher:rw /home/student/os_project/unit09/notes.txt # 6. ตรวจสอบค่า ACL ของไฟล์ getfacl /home/student/os_project/unit09/notes.txt
