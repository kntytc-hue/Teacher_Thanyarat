    Unit 04: CPU Scheduling Algorithms & Concepts

    หน่วยที่ 4 การจัดตารางเวลาซีพียู

    1. แนวคิดพื้นฐานเกี่ยวกับการจัดตารางเวลาซีพียู (CPU Scheduling Basics)

    ในระบบปฏิบัติการแบบ Multiprogramming เป้าหมายสูงสุดคือการทำให้ซีพียูทำงานอยู่ตลอดเวลา (Maximize CPU Utilization) เพื่อให้ใช้ทรัพยากรระบบได้อย่างคุ้มค่าที่สุด โดยวงรอบการทำงานของโพรเซสจะประกอบไปด้วยการสลับกันระหว่าง CPU Burst (ช่วงที่ประมวลผลคำสั่งในซีพียู) และ I/O Burst (ช่วงที่รอคอยการทำงานของอุปกรณ์อินพุต/เอาต์พุต)

    เมื่อซีพียูอยู่ในสถานะว่าง (Idle) ตัวจัดตารางเวลาระดับสั้น (Short-term Scheduler หรือ CPU Scheduler) จะทำการคัดเลือกโพรเซสที่อยู่ในสถานะ Ready ภายในหน่วยความจำหลัก เพื่อส่งมอบให้ไปประมวลผลบนซีพียูต่อไปผ่านกลไกที่เรียกว่า Dispatcher

    2. เกณฑ์ที่ใช้ในการวัดประสิทธิภาพ (Scheduling Criteria)

    การเลือกใช้อัลกอริทึมจัดคิวซีพียูที่เหมาะสม จำจำเป็นต้องพิจารณาเกณฑ์ชี้วัดประสิทธิภาพในด้านต่างๆ ดังต่อไปนี้:

    • CPU Utilization: เปอร์เซ็นต์เวลาที่ซีพียูทำงานจริง (ควรมีค่าสูงที่สุด)
    • Throughput: จำนวนโพรเซสที่ทำงานเสร็จสิ้นต่อหนึ่งหน่วยเวลา (ควรมีค่าสูงที่สุด)
    • Turnaround Time: เวลารวมตั้งแต่โพรเซสเข้าสู่ระบบจนทำงานเสร็จสิ้น (ควรมีค่าน้อยที่สุด)
    • Waiting Time: เวลารวมที่โพรเซสต้องนั่งรออยู่ใน Ready Queue (ควรมีค่าน้อยที่สุด)
    • Response Time: เวลาตั้งแต่ส่งคำร้องขอไปจนกระทั่งระบบเริ่มตอบสนองครั้งแรก (ควรมีค่าน้อยที่สุด)

    3. ประเภทของอัลกอริทึมจัดตารางเวลา (Scheduling Algorithms)

    อัลกอริทึมที่ใช้จัดคิวการทำงานแบ่งออกเป็นแบบหลัก ๆ ได้แก่แบบที่ยอมให้แทรกงานได้ (Preemptive) และแบบไม่ยอมให้แทรกงาน (Non-preemptive) โดยมีรูปแบบยอดนิยมดังนี้:

    First-Come, First-Served (FCFS)

    ใครมาก่อนได้รันก่อน เป็นแบบง่ายที่สุด แต่อาจเกิดปัญหาผลกระทบขบวนรถไฟ (Convoy Effect) เมื่อโพรเซสใหญ่ขวางโพรเซสเล็ก

    Shortest-Job-First (SJF)

    เลือกโพรเซสที่มีระยะเวลา CPU Burst สั้นที่สุดขึ้นมาทำก่อน ให้ค่าเฉลี่ยเวลาการรอคอยดีที่สุด แต่อาจเกิดสภาวะการอดตาย (Starvation)

    Round Robin (RR)

    ออกแบบมาเพื่อระบบ Time-sharing โดยเฉพาะ ทุกโพรเซสจะได้เวลาสลับกันรันเท่า ๆ กันตามคาบเวลาที่กำหนด (Time Quantum)

    4. ผังการเปรียบเทียบการจัดเวลาแบบ Preemptive vs Non-preemptive

    ความแตกต่างสำคัญของพฤติกรรมการควบคุมของระบบปฏิบัติการสามารถจำแนกออกได้ผ่านสิทธิ์ในการแย่งชิงทรัพยากร:

    เปรียบเทียบสิทธิ์การประมวลผล
    Non-Preemptive (ไม่สิทธิ์จัดสรรใหม่)
    โพรเซสจะยึดซีพียูยาวจนกว่าจะจบงานหรือปล่อยมือไปทำงาน I/O เอง
    Preemptive (ระบบมีสิทธิ์เข้าแทรกแทรง)
    OS สามารถสั่งหยุดโพรเซสกลางคันเพื่อสลับให้งานที่สำคัญกว่าทำได้ทันที

    5. Animation แอนิเมชันจำลองลำดับคิวและการประมวลผลของ CPU

    ตัวแบบจำลองนี้แสดงสถานการณ์ขณะที่ตัวจัดระบบทำการดึงงานจากแถวรอ (Ready Queue) เข้าสู่แกนประมวลผล (CPU Core) และขับออกเมื่อเสร็จสิ้นตามเกณฑ์เวลา

    นักศึกษาสามารถคลิกปุ่มรันตัวจำลองเพื่อดูการจัดสรรคิวของซีพียูแบบจำลอง:

    6. สภาวะอดตาย (Starvation) และการแก้ไขด้วยการเพิ่มอายุ (Aging)

    ในกลุ่มอัลกอริทึมที่ให้ความสำคัญกับระดับความสำคัญ (Priority) หรือขนาดงานสั้น (SJF) อาจเกิดวิกฤตที่โพรเซสซึ่งมีความสำคัญต่ำหรือมีขนาดงานใหญ่ต้องจมติดคิวอยู่อย่างนั้นยาวนานเนื่องจากมีงานใหม่ที่ดีกว่าแทรกตัวเข้ามาตลอดเวลา สภาวะนี้เรียกว่า Starvation (สภาวะอดตาย) วิธีแก้ไขที่นิยมคือการทำ Aging (การเพิ่มอายุ) โดยระบบปฏิบัติการจะทยอยเพิ่มลำดับความสำคัญให้โพรเซสเก่าที่รอนานตามเวลาที่ผ่านไป เพื่อการันตีว่าในที่สุดจะได้รับการประมวลผล
