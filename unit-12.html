    Unit 12: Multi OS Part 1

    หน่วยที่ 12 Multi OS ตอนที่ 1

    1. แนวคิดระบบปฏิบัติการคู่ (Dual Boot Concept)

    ระบบปฏิบัติการแบบ Dual Boot หรือระบบปฏิบัติการร่วม คือเทคนิคการติดตั้งระบบปฏิบัติการมากกว่าหนึ่งระบบลงบนเครื่องคอมพิวเตอร์เครื่องเดียวกัน (บนฮาร์ดดิสก์หรือ SSD ลูกเดียวกันหรือคนละลูกก็ได้) โดยผู้ใช้งานสามารถเลือกได้ว่าจะบูตเข้าสู่ระบบปฏิบัติการใดในขณะที่เปิดเครื่องคอมพิวเตอร์ขึ้นมาผ่านโปรแกรมจัดการการบูต (Boot Manager / Boot Loader) เช่น GRUB หรือ Windows Boot Manager

    การติดตั้งแบบ Dual Boot ระหว่างระบบปฏิบัติการ Windows และ Ubuntu Linux เป็นที่นิยมอย่างมากในหมู่นักศึกษาและนักพัฒนาซอฟต์แวร์ เนื่องจากช่วยให้สามารถใช้งานโปรแกรมสำนักงานและความบันเทิงบน Windows ควบคู่ไปกับการเขียนโปรแกรม การจัดการระบบเครือข่าย และการใช้งานเครื่องมือโอเพนซอร์สบน Linux ได้อย่างเต็มประสิทธิภาพโดยไม่ต้องใช้เครื่องคอมพิวเตอร์สองเครื่อง

    2. ความรู้พื้นฐานเกี่ยวกับ Windows และ Ubuntu

    Windows: เป็นระบบปฏิบัติการกรรมสิทธิ์ (Proprietary OS) ที่พัฒนาโดยบริษัท Microsoft มีส่วนติดต่อผู้ใช้แบบกราฟิก (GUI) ที่ใช้งานง่าย ได้รับความนิยมสูงสุดในตลาดผู้ใช้ทั่วไป การเล่นเกม และงานเอกสารสำนักงาน

    Ubuntu Linux: เป็นระบบปฏิบัติการเสรีและโอเพนซอร์ส (Open Source OS) ตระกูล Debian ที่มีความเสถียรสูง ปลอดภัยจากมัลแวร์ นิยมใช้เป็นเซิร์ฟเวอร์และแพลตฟอร์มสำหรับการพัฒนาซอฟต์แวร์สมัยใหม่ ปัญญาประดิษฐ์ (AI) และวิทยาการข้อมูล (Data Science)

    3. การเตรียมพาร์ทิชัน (Partition Preparation)

    ก่อนที่จะทำการติดตั้ง Ubuntu ควบคู่ไปกับ Windows ที่มีอยู่เดิม จำเป็นต้องมีการเตรียมพื้นที่ว่างบนฮาร์ดดิสก์ (Unallocated Space) โดยใช้เครื่องมือจัดการดิสก์ใน Windows (Disk Management) หรือเครื่องมือพาร์ทิชันภายนอก

    • การย่อขนาดพาร์ทิชัน (Shrink Volume): ทำการลดขนาดพาร์ทิชันหลักของ Windows (เช่น Drive C:) เพื่อเผื่อพื้นที่ว่างไว้อย่างน้อย 30 - 50 GB สำหรับการติดตั้ง Ubuntu
    • พื้นที่ว่าง (Unallocated Space): พื้นที่ที่ถูกกันไว้โดยยังไม่ฟอร์แมต เพื่อให้โปรแกรมติดตั้ง Ubuntu มองเห็นและนำไปสร้างพาร์ทิชันย่อยในขั้นตอนต่อไป

    4. การสร้าง USB Bootable ด้วยโปรแกรม Rufus

    การติดตั้ง Ubuntu จำเป็นต้องใช้แฟลชไดรฟ์ USB (ความจุอย่างน้อย 4GB - 8GB) เป็นสื่อในการติดตั้ง โดยมีขั้นตอนและเครื่องมือสำคัญคือ Rufus ซึ่งเป็นโปรแกรมโอเพนซอร์สขนาดเล็กบน Windows ที่ใช้สำหรับแปลงไฟล์ ISO ของระบบปฏิบัติการให้เป็น USB Bootable

    1. ดาวน์โหลดไฟล์อิมเมจระบบปฏิบัติการ Ubuntu ในรูปแบบไฟล์ .iso จากเว็บไซต์ทางการของ Ubuntu
    2. เชื่อมต่อ USB Flash Drive เข้ากับเครื่องคอมพิวเตอร์ (ข้อมูลใน USB จะถูกลบทั้งหมด)
    3. เปิดโปรแกรม Rufus เลือก Device ให้ตรงกับ USB ที่ต้องการ
    4. ในส่วน Boot selection ให้เลือกไฟล์ ISO ของ Ubuntu ที่ดาวน์โหลดมา
    5. กำหนด Partition scheme เป็น GPT (สำหรับเมนบอร์ดระบบ UEFI สมัยใหม่) และ File system เป็น FAT32
    6. คลิกปุ่ม START และรอจนกระทั่งกระบวนการเขียนข้อมูลลง USB เสร็จสมบูรณ์

    5. การตั้งค่า BIOS / UEFI Boot Setting

    เมื่อสร้าง USB Boot เรียบร้อยแล้ว ต้องทำการรีสตาร์ทเครื่องคอมพิวเตอร์และเข้าสู่หน้าจอตั้งค่าระบบพื้นฐานของเมนบอร์ด (BIOS หรือ UEFI) โดยการกดปุ่มลัด (เช่น F2, F12, Del, หรือ Esc ตามยี่ห้อเมนบอร์ด)

    • ปิดระบบความปลอดภัย Secure Boot (ในบางกรณีเพื่อป้องกันปัญหาการบูตไดรเวอร์ของ Linux)
    • ตั้งค่าลำดับการบูต (Boot Priority) ให้ USB Flash Drive อยู่ในลำดับแรก (USB Boot / UEFI USB)
    • บันทึกการตั้งค่า (Save & Exit) เพื่อให้เครื่องบูตเข้าสู่หน้าจอติดตั้ง Ubuntu จาก USB

    6. ขั้นตอนการติดตั้ง Ubuntu และการจัดการแบบ Something Else

    เมื่อบูตเข้าสู่หน้าจอติดตั้งของ Ubuntu จะพบกับหน้าต้อนรับ (Welcome Screen) ให้เลือกภาษาที่ต้องการและรูปแบบคีย์บอร์ด จากนั้นเข้าสู่หน้าจอเชื่อมต่ออินเทอร์เน็ตและการเลือกรูปแบบการติดตั้ง (Normal installation / Minimal installation)

    หัวใจสำคัญของการทำ Dual Boot คือหน้าต่าง Installation Type ซึ่งจะมีตัวเลือกให้ติดตั้ง Ubuntu ควบคู่ไปกับ Windows อัตโนมัติ หรือเลือกตัวเลือกขั้นสูงที่เรียกว่า "Something Else" (อย่างอื่น) ซึ่งแนะนำสำหรับผู้เรียนระดับ ปวส. เพื่อความปลอดภัยของข้อมูลและอิสระในการกำหนดพาร์ทิชันเอง:

    • Root Partition (/): กำหนดจุดเชื่อมต่อเป็น / ใช้ระบบไฟล์ Ext4 สำหรับเก็บไฟล์ระบบปฏิบัติการและโปรแกรมทั้งหมด (แนะนำพื้นที่อย่างน้อย 25-30 GB ขึ้นไป)
    • Swap Area: พื้นที่หน่วยความจำเสมือนบนดิสก์ (แนะนำให้มีขนาดเท่ากับหรือใกล้เคียงกับ RAM จริง เช่น 4GB หรือ 8GB)
    • EFI System Partition: สำหรับเครื่องระบบ UEFI ให้ใช้พาร์ทิชัน EFI ที่มีอยู่แล้วจาก Windows ห้ามลบเด็ดขาด (เลือกให้ติดตั้ง Boot loader ลงบนพาร์ทิชัน EFI นี้)
